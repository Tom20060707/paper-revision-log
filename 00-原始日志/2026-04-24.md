## 算法/公式变化

**新增 Φ_prog 显式公式（Section 4.4，原来只有文字描述）：**
```
Φ_prog(s) = Σ w_i · c_i(s)，c_i(s) ∈ [0,1]
```
c_i(s) 是第 i 个子任务完成度（门开角、对象到目标距离归一化值等）。

**新增 m_t 门控乘子（Section 4.4）：**
```
r'_t = r_t + λ_prog · m_t · (γ·Φ_prog(s_{t+1}) - Φ_prog(s_t))
```
m_t=0 在 Protect 和 Normal 模式下自动抑制 shaping，仅 Push 模式 m_t=1 激活。

**原因：** 原公式缺少 m_t，会让审稿人误以为 PBRS 在所有训练阶段都激活，与三模式互斥逻辑矛盾。

## 新洞察

1. **PBRS 的核心贡献层级**：在没有 Kitchen 实验数据的情况下，正确的声明方式是设计与形式化贡献而非实验验证贡献。两者之间最多差 10% 的接受概率（从 ~52% 到 ~62%），但边界不清会直接导致拒稿。

2. **Fig. 4 拆分的必要性**：locomotion 和 manipulation 的 FTC 分布形态根本不同（locomotion 集中于 0.42–0.55，manipulation 更分散且 Mode B 主要出现在 manipulation），硬塞一张图会掩盖这个结构性差异，审稿人很难理解 Mode A 为什么在某些任务上可行而另一些不行。

3. **图的颜色语义统一是论文说服力的乘法效应**：蓝=Vanilla、橙=TC-PPO miscalibrated/Mode B、绿=TC-PPO soft/Mode A，一旦在第一张图建立，读者看后续每张图都不需要重新学习图例，认知负担大幅降低。
