# 2026-02-18 研究日记

## 今日主线

今天主要推进了 [[TC-PPO Algorithm]] 论文 `nips2026.tex` 的 NeurIPS 风格修订，重点不是新增实验结果，而是**降低论文叙事风险、补齐实验协议、提升图表质量与可复现表达**。

核心变化有三条：

1. **重新定位论文贡献**  
   将论文从容易被误读为“HumanoidBench 全局 SOTA”的叙事，收紧为“稳定性优先的、可扩展的 model-free PPO 改进方案”。  
   当前已有结果仍保留：  
   - 8.2M steps：TC-PPO `842.1±18.3`  
   - 10M steps：TC-PPO `859.3±15.5`  
   但不再暗示超过 TD-MPC2、DreamerV3 等 model-based SOTA。

2. **把 [[FTC Metric]] 的因果声称降级为更可信的 temporal alignment**  
   原先“FTC 可预测 collapse / FTC precedes degradation”一类表述被视为风险点，因为尚缺少 FTC 特异性对照实验。今天将其改成更保守的“时间对齐分析”，并在附录中列出后续验证协议。

3. **大幅重构图表与附录实验计划**  
   修复了两张核心图的排版问题：8 子图多任务网格图、Reward/FTC 训练动态图；同时将附录中的 planned validation 扩展为 A–E 实验协议，并加入实现模板，如 `compute_ftc`、`FTCTracker`、结构化 JSONL 日志和 `detect_collapse`。

## 关键对话与决策

### 1. 收紧 SOTA 相关表述

对论文与 HumanoidBench SOTA 进行了对比后，确认一个关键风险：当前 TC-PPO 的 10M 步结果 `859.3±15.5` 不能被表述为超越 HumanoidBench 官方 SOTA。

对话中参考到的外部基准区间包括：

| 方法 | H1 Walking Reward | 备注 |
|---|---:|---|
| TD-MPC2 | 约 `920–950` | model-based SOTA，待以官方表格核实 |
| DreamerV3 | 约 `880–910` | model-based |
| DrQ-v2 | 约 `850–870` | model-free / visual RL 相关基线，待确认具体设置 |
| TC-PPO 10M | `859.3±15.5` | 本文当前结果 |

因此做出决策：

- 不再使用可能被解读为“全局 SOTA”的语言。
- 摘要、引言和主结果处强调：本文目标是缩小 parallel PPO 与强 model-based 方法之间的 sample-efficiency/stability gap。
- 贡献重心改成：
  - training stability；
  - parallel PPO scalability；
  - contact-aware adaptive regularization；
  - FTC 作为机制解释与控制信号，而不是未经充分验证的因果预测器。

相关双链：[[HumanoidBench]]、[[TC-PPO Algorithm]]、[[FTC Metric]]

### 2. 将 FTC 因果叙事改为保守表述

今天明确识别到论文中最危险的科学断言之一：

> FTC often precedes performance degradation  
> FTC as early proxy for impending gait instability

这些表述目前证据不足，因为还没有完成如下关键对照：

- 随机打乱 FTC 信号；
- 用无关代理信号替代 FTC；
- 固定调度 vs FTC 调度；
- 不同阈值/滞后窗口敏感性；
- collapse 事件检测的稳健性。

因此今日决策是：

- 正文中避免“预测”“因果”强表述；
- 改成 temporal alignment / association；
- 在 Limitations 和 Appendix 中明确列为待补实验，而不是假装已有结论。

这让论文在审稿中更稳健：即便审稿人质疑 FTC 是否“真正特殊”，当前稿件也已主动承认缺口，并给出补实验方案。

相关双链：[[FTC Metric]]、[[Causal Claims in RL Papers]]、[[Mode A-B-C Taxonomy]]

## 新洞察

### 1. TC-PPO 当前最强卖点不是“SOTA”，而是“稳定性机制”

今天最重要的判断是：  
[[TC-PPO Algorithm]] 当前不应该与 TD-MPC2 / DreamerV3 在“最高 reward”上硬碰硬，而应该强调它在 parallel PPO 框架下对 contact-induced instability 的机制化处理。

更合适的论文卖点是：

- 仍保留 PPO 的并行扩展性；
- 通过 [[FTC Metric]] 监控步态接触节律；
- 在 FTC 退化时调节 entropy/KL 正则；
- 减少训练 collapse 或不稳定区间；
- 给出比 vanilla PPO 更稳定的 model-free 训练路径。

这比“追求全榜单 SOTA”更可信，也更容易经受审稿质疑。

### 2. FTC 需要“特异性验证”，否则只能算相关信号

FTC 的理论动机来自混合动力系统、Poincaré 映射与极限环稳定性，这给了它很好的解释基础。但从论文证据链看，仍缺少一个核心环节：

> FTC 的效果是否优于任意 adaptive signal？

如果打乱 FTC 或换成随机信号后 TC-PPO 仍然有效，那么 FTC 的特殊性会被削弱。  
所以后续最关键的补实验不是再画更多曲线，而是验证：

- FTC-shuffled；
- random proxy；
- fixed schedule；
- no FTC；
- FTC threshold sweep。

这会直接决定 [[FTC Metric]] 是“机制性指标”还是“经验性启发式信号”。

### 3. 图表设计影响论文可信度

今日图表重构暴露一个问题：  
即使实验数据不变，图表如果遮挡、重叠、双轴混乱，也会显著削弱论文可信度。

尤其是 Reward 与 FTC 原先如果放在双轴图中，容易让读者误以为两者尺度可直接比较。拆成上下共享 x 轴后，因果含义更清楚，也更符合论文当前的保守叙事：展示 temporal alignment，而不是暗示严格因果。

### 4. “计划实验”必须和“已有结果”明确隔离

今天反复强调：附录可以加入实验计划和 protocol，但不能填入未跑数据。  
这对公开仓库和论文都很重要：

- 已有结果：可以写数值；
- 缺失验证：写 protocol / template / camera-ready extension；
- 不确定内容：标记待确认；
- 不能用 AI 生成未验证的实验结论。

这条原则对后续所有论文自动化写作都应保持。
