# 2026-04-20 研究日志

今天的主线从“如何向人形机器人实习导师解释自己的研究经验”逐步推进到 [[TC-PPO Algorithm]] 的下一阶段实验与算法叙事重构。

### 2. 实验主线：从“补鲁棒性实验”升级为“扰动驱动算法进化”

最初的下一步实验建议是做 force disturbance，对 `stair` / `slide` 加外部扰动，用于验证现有 TC-PPO 在扰动环境下的鲁棒性。但用户指出：既然任务难度提升，那么算法也应同步进化，而不是把算法改进拆到下一篇论文中。

今天形成的新主线是：

> 标准任务中，当前 [[TC-PPO Algorithm]] 解决基础崩溃问题；  
> 扰动任务中，接触不一致性加剧，现有反应式门控可能不够；  
> 因此任务难度提升自然驱动算法进化；  
> 算法进化后再挑战更强扰动任务，形成正向闭环。

这一叙事比单纯“补一个 robustness 实验”更强，因为它同时服务于：

- 当前论文缺少“Vanilla 明显崩溃、改进算法明显救活”的强证据问题；
- [[Sim-to-Real Robustness]] 的合理铺垫；
- 算法改进的动机解释；
- 审稿人对 necessity 的疑问。

### 3. 算法组件取舍：CUSUM + GAE reweighting 成为核心组合

今天讨论了多个候选算法改进：

1. **Lyapunov-inspired FTC stability margin**  
   用 [[FTC Metric]] 的历史趋势项提前预判不稳定，而不是等 FTC 跌破阈值后才触发门控。

2. **GAE variance re-weighting**  
   在 advantage estimation 中用 FTC 对样本贡献进行加权，降低接触混乱时间步对梯度的污染。

3. **Hybrid-mode-aware entropy schedule**  
   根据 contact-switch frequency 动态调整 KL 或熵相关项。

4. **Critical slowing down / early warning signals**  
   借鉴生态学、气候系统中的临界转变预警，用 FTC 序列的滑动窗口自相关与方差作为崩溃前兆。

5. **CUSUM 统计过程控制**  
   将 FTC 看作过程质量指标，用累积和控制图检测缓慢分布漂移，比单点阈值更适合扰动环境。

6. **CBF / Kalman / limit-cycle spectral analysis 等跨学科扩展**  
   作为更远期候选，但当前论文中可能过重。

最后形成的较清晰决策：

- 当前论文不推荐放入过多复杂跨学科机制；
- Multiscale entropy、CBF、完整谱半径分析暂不作为当前主线；
- **核心应是 CUSUM + GAE reweighting**：
  - CUSUM：负责“何时干预”，替代或增强硬阈值；
  - GAE reweighting：负责“干预后如何更新”，从 loss / advantage 层面降低 corrupted samples 影响；
- Lyapunov margin 可作为 CUSUM 的输入增强或消融项，而不宜与 CUSUM 并列为主贡献，因为二者都偏向“何时干预”，职责有重叠；
- Hybrid entropy schedule 当前价值较弱，容易与门控逻辑重复，暂不推荐作为核心。

这一点修正了另一版建议中“CUSUM + Lyapunov 为主、GAE 作为辅助”的观点。今天的判断是：如果只保留两个核心，应该保留 CUSUM 与 GAE，而不是 CUSUM 与 Lyapunov。

### 4. 实验资源与训练流程：从“写代码”转向“小而精训练”

用户明确表示代码可以自己写，不需要把太多时间留给代码开发，因此实验规划应缩短 Phase 0，增加训练时间，但不能因费用压力扩大过多。由此今日规划倾向于：

- 用本地/笔记本显卡做 sanity check 和小规模调试；
- 用服务器做正式扫描与关键对比；
- 选择小而精任务，而不是全任务大规模铺开；
- 优先选择已有 [[Mode A-B-C Taxonomy]] 基础、扰动下最可能拉开差距的任务。

当前任务选择仍以 `stair` 和 `slide` 为主，理由是：

- 已有 Mode A 基础；
- 对接触扰动敏感；
- 适合制造 Vanilla PPO 崩溃、增强版 TC-PPO 稳定恢复的对比；
- 成本可控。

### 5. 训练流程解释：明确“一个完整训练流程”包含什么

今天详细解释了一个完整训练流程，不再停留在代码层面，而是用大白话说明各阶段：

1. **初始化**
   - 加载 HumanoidBench / MuJoCo 环境；
   - 初始化 Actor、Critic；
   - 初始化 rollout buffer；
   - 初始化 FTC 计算器；
   - 初始化 CUSUM 状态。

2. **Warmup 阶段**
   - 机器人先运行一段时间，只观测不学习；
   - 记录正常情况下 FTC 分布；
   - 用于确定 CUSUM 基准值、门控阈值等参考量；
   - 这一阶段的意义是建立“正常接触时序”的基线。

3. **正式训练阶段**
   - 每一步：策略根据状态输出动作，环境执行动作并返回奖励与新状态；
   - 同时从接触信息计算 FTC；
   - CUSUM 根据 FTC 偏移判断是否应触发门控；
   - 每收集一批 rollout 数据后进行 PPO 更新；
   - GAE reweighting 在 advantage 计算阶段降低低 FTC 样本权重；
   - 更新后丢弃旧 rollout，继续下一轮收集。

4. **结果保存**
   - 保存 reward 曲线、FTC 轨迹、CUSUM 触发次数、模型权重、summary 文件等。

核心理解：

> PPO 是“收集一批数据 → 更新一次 → 丢掉旧数据 → 再收集”的 on-policy 循环；  
> TC-PPO 的各类机制就是嵌入到“环境扰动、FTC 计算、门控判断、advantage 计算、PPO 更新”这几个位置中。

### 6. 学科知识如何落到脚本修改

今天明确了“数学、物理、自控、生物力学、RL 理论”不是抽象口号，而是分别对应脚本中的不同位置：

- **Sim-to-Real / domain randomization**
  - 改环境层；
  - 加外力脉冲、摩擦随机化、action noise、observation noise；
  - 目标是让训练环境更接近真实硬件扰动。

- **生物力学 / 接触动力学**
  - 改 FTC 计算层；
  - 用接触时序、步态节律、接触间隔变化描述运动稳定性。

- **数学 / 物理趋势预判**
  - 改 FTC 信号增强层；
  - 例如 Lyapunov-inspired margin、critical slowing down 预警信号等。

- **自控 / 统计过程控制**
  - 改 gate 判断层；
  - 用 CUSUM 代替简单硬阈值，检测分布漂移与累积异常。

- **强化学习理论**
  - 改 advantage / loss 计算层；
  - 用 GAE reweighting 减少接触异常样本对策略梯度的污染。

这使“跨学科算法改进”落到了具体可实现位置，而不是停留在论文包装。

## 新洞察

### 1. “任务升级 + 算法升级”比“单独补鲁棒性实验”更有论文力量

今天最重要的认知变化是：扰动实验不应只是作为 robustness appendix，而应成为算法演化的驱动力。

原始逻辑：

> 标准任务上 TC-PPO 有效；  
> 再加一个扰动实验证明鲁棒性。

新的逻辑：

> 标准任务暴露基础崩溃；  
> TC-PPO 解决第一层问题；  
> 扰动任务进一步放大接触不一致性；  
> 原有反应式门控出现局限；  
> 因此引入 CUSUM + GAE reweighting；  
> 增强版算法在更难任务中恢复稳定。

这条因果链更完整，也更能回应“为什么需要新算法”的审稿问题。

### 2. CUSUM 与 Lyapunov margin 不应同时作为并列核心

今天澄清了一个组件职责问题。CUSUM 和 Lyapunov margin 都偏向于“更早发现不稳定”，一个看累积漂移，一个看趋势变化。如果两者并列作为核心贡献，容易显得重复。

更干净的结构是：

- CUSUM：主检测器；
- Lyapunov margin：可作为输入信号增强；
- GAE reweighting：更新层治疗机制。

这使方法从“多个检测器堆叠”变成“检测 + 治疗”的正交组合。

### 3. GAE reweighting 是当前算法进化中最不可替代的一环

虽然 GAE reweighting 可能引入 bias 风险，但它解决的是 CUSUM / Lyapunov 无法解决的问题：低质量接触样本已经进入 rollout 后，如何避免其污染梯度。

因此它不应被轻易降级为“可选小组件”。更合理的处理是：

- 加权强度需要保守；
- 可使用 clamp 限制最低权重，避免过度压制样本；
- 在 ablation 中单独验证其贡献；
- 在论文中解释为“降低 corrupted contact samples 对 advantage estimation 的影响”，而不是声称完全消除噪声。

### 4. 跨学科元素必须映射到代码位置，否则容易变成包装

今天把数学、物理、自控、生物力学和 RL 理论分别映射到训练脚本的不同模块，这是一个很重要的工程化理解：

- 环境层：扰动；
- 接触信号层：FTC；
- 预警层：趋势、EWS；
- 控制层：CUSUM / gate；
- 学习层：GAE / PPO loss。

这个映射能帮助后续写论文方法部分，也能帮助自己实现代码时不混乱。

### 5. 对外表达研究经验时，“少细节但保留逻辑”更重要

上午的实习回复整理说明：在匿名投稿阶段，对外介绍论文不能靠堆公式和实验数字，而应展示：

- 自己发现了什么问题；
- 为什么这个问题重要；
- 用什么类型的方法处理；
- 做过怎样规模和类型的实验；
- 掌握哪些工具与能力。

这对后续联系导师、博士生、实验室实习都有直接价值。

### 风险

1. **扰动强度不好选**
   - 太弱：Vanilla 不崩，亮点不足；
   - 太强：所有算法都崩，无法证明增强版有效；
   - 需要 Phase 1 扫描找到临界区间。

2. **GAE reweighting 可能引入 bias**
   - 需要控制权重范围；
   - 不宜让低 FTC 样本完全消失；
   - 应在 ablation 中验证是否稳定提升。

3. **算法组件过多导致主线发散**
   - 当前论文应克制；
   - CUSUM + GAE 为主；
   - Lyapunov margin 作为增强或消融；
   - MSE、CBF、Kalman、limit-cycle spectral analysis 暂不进入主贡献。

4. **训练成本压力**
   - 不宜全任务、多噪声、多 seed 同时展开；
   - 先做 `stair` / `slide` 小而精；
   - 本地只做 sanity check，服务器跑正式结果。

5. **对外文件需注意保密**
   - 实习回复中不能出现完整公式、具体未公开实验数字、匿名投稿论文细节；
   - 可以保留研究方向和能力描述。
