## 新洞察

1. **研究日志自动化本身也是研究基础设施的一部分**  
   对 [[TC-PPO]] 这类持续迭代项目来说，每日研究日志不仅是记录结果，更是保证实验、论文、代码与决策可追溯的机制。

2. **需要区分“存在历史日志”和“今日日志已生成”**  
   今天明确暴露了一个容易混淆的问题：仓库中已有 `2026-04-30.md` 并不代表 `2026-05-01.md` 已完成。后续应始终检查当前日期文件是否存在。

3. **日志生成必须基于可验证来源**  
   如果自动脚本失败，正确做法是报告错误输出，而不是为了“补齐日期”编写无来源内容。这对论文修改记录和研究复现都很重要。

4. **自动化流程应同时验证写入、镜像、提交和推送**  
   日志同步不是只生成一个 Markdown 文件，还应确认：
   - 是否写入 Obsidian 主库；
   - 是否镜像到论文修改日志目录；
   - 是否 commit；
   - 是否 push 到 GitHub 远端。
