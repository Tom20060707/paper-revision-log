# 2026-02-15 研究日记

## 今日主线

今天的主线是把 `nips2026.tex` 从“方法草稿 + 大量占位 + 潜在编译问题”的状态，推进到一个更接近 NeurIPS 投稿标准的可编译论文骨架：

1. **论文文本去幻觉与审稿友好化**：围绕 [[TC-PPO Algorithm]]、[[FTC Metric]] 的核心方法，删除未经实验证实的 SOTA、百分比提升、零崩溃等强结论，把实验结果统一改成 `TBD` 或需由真实日志回填的表格。
2. **LaTeX 编译稳定化**：按报错逐项修复 `nips2026.tex` 中的引用、Unicode、caption、标题、图文件名、宏与模板 fallback 问题，最终确认完整编译链可跑通。
3. **补齐论文图资产生成流程**：新增 `generate_paper_figures.py` 与 `build_paper_assets.sh`，为论文生成缺失的图文件，避免正文出现大块空白占位。
4. **参考文献切换到 `ref.bib` 全量导入**：通过 `\nocite{*}` 与 `\bibliography{ref}` 让 `ref.bib` 中 40 篇论文全部进入 References，并验证 `nips2026.bbl` 中 `\bibitem` 数量为 40。
5. **按 NeurIPS 结构要求继续打磨论文**：新增 `Problem Statement`、补充 TC-PPO 与现有 adaptive PPO 的区别、增强算法伪代码、清理非学术表述，并补充 `Data Availability`。

## 关键对话与决策

### 1. 从“漂亮结论”转向“可验证结论”

上午首先对 `nips2026.tex` 做了面向 NeurIPS 审稿的风险清洗。核心决策是：

- **保留方法主线**：继续以 [[FTC Metric]] 与 [[TC-PPO Algorithm]] 为论文核心。
- **删除无法本地核验的实验数字和强断言**：
  - 不再写未经日志支持的 SOTA。
  - 不再写未经核验的百分比提升。
  - 不再写“no collapse / zero collapse”等绝对稳定性表述。
- **将实验结果区改为真实日志回填协议**：
  - 结果表保留结构，但数值改为 `TBD`。
  - 明确只能用真实训练日志回填。
  - 将统计策略写成可执行规范，而不是先给结论。

这一步的研究意义是：论文从“看起来完整但容易被 technical soundness / reproducibility 质疑”，转向“结论克制、证据链待补但审稿风险更低”的版本。

### 2. 编译优先级高于表述优化

中午用户明确要求“不能有语法错误，确保成功编译”。因此当天第二个关键决策是：先把论文修到稳定编译，再谈进一步内容润色。

已修复的问题包括：

- 移除或替换 `ref.bib` 中不存在的 citation key。
- 清理高风险 Unicode 字符：
  - `±` 改为 `$\pm$`
  - `×` 改为 `$\times$`
  - `≈` 改为 LaTeX 写法
  - `↑/↓` 改为 LaTeX 可编译表达
  - `10⁶` 改为 `$10^6$`
- 修复 caption 中未转义或错误转义的 `%`：
  - 其中 `95\\%` 被修正为 `95\%`。
- 修复小节标题中的 `&`：
  - `Training Dynamics & ...` 改为 `Training Dynamics \& ...`。
- 补上双轴图缺失的 `\end{axis}`。
- 将 `\safeincludegraphics` 中带 `_` 的图文件名改成连字符版本，避免宏展开时报 `Missing $ inserted`。
- 将 `natbib` 放入 fallback 分支，避免在有 `neurips_2024.sty` 的 Overleaf 环境里潜在 package option clash。
- 当本地缺少 `neurips_2024.sty` 时，允许回退到 `geometry+natbib`，保证本地也能编译。

验证结果：

- `pdflatex -interaction=nonstopmode -halt-on-error nips2026.tex` 成功。
- 完整链路 `pdflatex -> bibtex -> pdflatex -> pdflatex` 成功。
- 仍有排版 warning，例如 `Overfull \hbox`，但没有致命语法错误。

## 新洞察

### 1. TC-PPO 论文当前最重要的瓶颈不是“写得不够强”，而是“证据闭环不够硬”

今天反复出现的核心问题是：论文文字、图和 references 可以被修到很完整，但真正决定 NeurIPS 审稿说服力的仍是：

- 真实训练日志；
- 多 seed 统计；
- collapse case 的可复现实证；
- sample efficiency 曲线；
- ablation 是否与 [[Mode A-B-C Taxonomy]] / [[FTC Metric]] 的理论动机一致。

因此，当前版本应被视为“可编译、结构完整、可回填真实结果的论文容器”，而不是已经完成实证闭环的最终稿。

### 2. “去幻觉”不是削弱论文，而是在建立可审稿性

删除 SOTA、百分比提升、零崩溃等强断言后，论文表面上更保守，但实际上更符合顶会审稿逻辑：

- 审稿人更容易接受“明确协议 + 待验证表格 + 真实日志回填”。
- 比未经证明的强结论更不容易触发 technical soundness 质疑。
- 对 [[TC-PPO Algorithm]] 这类稳定性导向方法来说，可信的失败案例和 collapse 诊断甚至比单纯 reward 提升更重要。

### 3. 图像资产需要区分“示意图”和“实验图”

今天生成的图解决了论文排版空白问题，但必须在知识库中明确分类：

- `paperbanana-framework.pdf` 更偏方法流程示意图。
- `visuomotor.pdf`、`collapse-case-study.pdf`、`contact-raster.pdf` 若未从真实日志生成，就只能作为模板或占位风格图，不能当作最终实验结果证据。

后续应尽量让这些图从真实日志自动生成，避免“看起来像实验图但不是实验数据”的审稿风险。

### 4. 全量 `ref.bib` 导入有利于完整性，但可能降低引用精准度

`\nocite{*}` 能保证 40 篇文献全部显示，适合当前草稿整合阶段。但最终投稿时需要检查：

- 是否有未在正文讨论的文献被无理由列入；
- related work 是否覆盖每类引用；
- 是否存在“堆 references”但没有分析差异的问题。

最终版本可能应从“全量导入”切换为“正文实际引用 + 少量关键补充”。

- **实验数字仍未闭环**：论文结构已经完整，但 `TBD` 结果仍需真实数据支撑。

- **pipeline 脚本不可用**：两个 pipeline 文件存在缩进错误，影响可复现叙事。
