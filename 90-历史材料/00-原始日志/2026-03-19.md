# 2026-03-19 研究日记

## 核心进展

今天主要围绕人形机器人长时程、接触丰富任务中的论文摘要进行打磨，重点不是新增实验，而是明确摘要中哪些主张可以写、哪些必须留空等待实验支撑。

当前摘要的核心设定是：提出一个面向 humanoid control / manipulation 的 **Contact-Aware Multimodal HRL** 框架，用层级强化学习将宏观规划与微观平衡控制解耦。摘要中包含四类贡献：

1. **方法指标**：提出 [[FTC Metric]]，即 Foot-contact Temporal Consistency，用于刻画足端接触区间的时间一致性，并作为训练稳定性/梯度不稳定性的可解释信号。
2. **算法机制**：设计 TC-SAC，通过实时 FTC 分数对 replay buffer 中可能导致不稳定、高方差的 transition 进行降权。
3. **系统架构**：构建多模态 HRL 系统，高层使用 world-model-based policy（DreamerV3），结合 late-fusion perception，包括 128-d visual latents 与 448-d tactile sensing，并向冻结的低层 TC-SAC policy 输出空间 setpoints。
4. **实验验证**：计划在 HumanoidBench 等长时程接触任务中验证，但具体实验数字不能编造，需要以括号或占位形式保留。

今天最重要的决策是：**摘要可以强调问题、方法、架构和预期评估维度，但所有性能提升、成功率、样本效率等数字必须留空，不能虚构。**

对话中的原始摘要认为，model-free 与 off-policy RL 方法，如 PPO、SAC，虽然可扩展到 humanoid control，但在长时程、接触丰富的 manipulation 任务中容易出现 catastrophic training collapse 与 sample inefficiency。摘要进一步指出，flat policy 难以同时优化高维平衡控制与复杂操作，因此在 HumanoidBench 这类 demanding benchmark 中容易出现收敛失败。

从研究叙事上看，摘要已经具备较强的论文雏形：问题明确、贡献分层清楚、指标—算法—架构—实验四段式完整。但风险也很明显：如果实验尚未完成，摘要中过度肯定的性能结论需要降级为“we evaluate / we show with placeholders”或保留占位，避免形成不可验证主张。

当天没有产生新的实验数据，也没有确认任何具体成功率、sample efficiency 提升倍数或 benchmark 分数。

## 新洞察

1. **摘要的可信度比“听起来完整”更重要**  
   当前摘要已有较强的论文包装能力，但实验结果尚未确认时，最容易出问题的是把“方法动机”写成“已验证结论”。今天明确了原则：可以写机制假设和实验计划，不能编造性能数字。

2. **[[FTC Metric]] 的定位需要更严谨**  
   FTC 被描述为“mechanistically interpretable signal aligned with hybrid-system stability principles”，用于量化 foot-contact interval regularity，并预测 gradient instability。这个定位很有价值，但后续必须用实验或分析验证：
   - FTC 是否真的与训练崩溃相关；
   - FTC 与 gradient variance / policy update instability 的相关性如何；
   - FTC 是在线降权信号，还是诊断指标，或两者兼具。

3. **层级解耦是摘要中最清晰的主线**  
   将 humanoid manipulation 分成：
   - 高层：world-model / multimodal planning；
   - 低层：contact-aware balance and motor control；  
   这个叙事比单纯“提出一个新 RL 算法”更有说服力，也更适合长时程接触任务。

4. **算法命名存在潜在风险**  
   当前摘要写的是 TC-SAC，但知识库重点可能围绕 [[TC-PPO Algorithm]]。如果实际项目中两者都存在，需要建立清晰关系：
   - TC-SAC 是否是 off-policy 版本；
   - TC-PPO 是否是 on-policy 版本；
   - [[FTC Metric]] 是否是二者共享的 temporal-consistency weighting signal；
   - 最终论文主线究竟选哪一个算法名。

5. **多模态数字可保留，但性能数字不能补**  
   来源中明确出现了 128-d visual latents 与 448-d tactile sensing，这些是架构级具体信息，可以保留。相比之下，任何实验收益数字均未出现，必须留空。
