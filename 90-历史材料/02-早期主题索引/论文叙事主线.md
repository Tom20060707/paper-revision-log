# 论文叙事主线

## 当前主线

当前论文不应再写成单纯的 **TC-PPO performance booster**，也不应只写成 **FTC/CTS 指标论文**。

更稳的主线是：

> 人形机器人接触域中的一致性与稳定性研究：接触一致性负责崩溃前诊断，接触域稳定性是训练和策略改进的目标。

## 叙事演化

### 早期版本：TC-PPO 性能论文

最早的表达接近：

```text
TC-PPO improves PPO on contact-rich humanoid tasks.
```

问题是 Mode A 只在部分任务上成立，Mode B/C 也很关键。如果继续写成普遍性能提升，会被负结果反噬。

### 中期版本：FTC 诊断框架

随后论文转向：

```text
FTC diagnoses PPO collapse before reward failure.
```

这一阶段的优势是诚实，也能解释负结果；但风险是 FTC 太像足部/周期步态指标。

### 后期版本：CTS-PPO

4 月 28 日之后，主线变成：

```text
Contact Timing Stability-Guided PPO
```

这比 FTC 更宽，也更贴近高维人形控制中的形态/接触不稳定性。

但仍需注意：如果核心公式仍是 `1 - CV²`，它只能作为准周期支撑接触实例，不能通吃非周期任务。

### 当前版本：CSC + 接触域一致性与稳定性

当前最推荐写法：

```text
Contact-Surface Consistency diagnoses contact-domain degradation.
PPO stabilization mechanisms suppress or recover from contact-domain instability.
```

中文逻辑：

```text
接触面一致性 -> 崩溃前预警
接触域稳定性 -> 崩溃抑制/稳定恢复
```

## 推荐表达

1. 高维人形 PPO 的困难不仅是动作维度大，还包括接触域容易退化。
2. 接触一致性是可观测诊断信号，用于发现接触域是否偏离稳定参考模式。
3. CTS/FTC 是支撑接触场景下的已验证实例。
4. CSC 是更一般的接触面一致性框架，适合解释非周期和阶段化任务。
5. Mode A/B/C 是稳定域状态分类，不是任务好坏分类。
6. PBRS/Kitchen 更适合写作边界分析或平台期推进机制，不应写成已解决操作任务。

## 不推荐表达

- “TC-PPO universally improves HumanoidBench performance.”
- “FTC/CTS explains all manipulation tasks.”
- “`1 - CV²` is the universal contact metric.”
- “PBRS solves kitchen-class tasks.”
- “Mode B/C are failed experiments to hide.”
- “Hand/object/tactile CTS has already been validated.”

## 负结果的用法

Mode B/C 应主动保留：

- Mode B 可以说明固定阈值、自适应阈值和接触面集合之间的边界；
- Mode C 可以说明“接触稳定”和“任务成功”不是一回事；
- Kitchen 可以说明非周期、阶段化、稀疏奖励任务需要更一般的 CSC 或任务进度机制。

负结果不是论文污点，而是接触域框架成立的证据来源。
