# 2026-02-17 研究日记

## 今日主线

今天主要推进了 [[TC-PPO Algorithm]] 论文 *H1-Scalable: TC-PPO, Temporal-Consistency Guided Stabilization for Parallel PPO in Humanoid Control* 的 NeurIPS camera-ready 级别可视化与 LaTeX 结构重构。

核心工作不是新增实验，而是把已有实验结果、图表、算法伪代码和叙事顺序整理成更接近顶会审稿预期的形式：  
- 以“零数据幻觉”为硬约束，只使用论文中已经报告的数值、均值、标准差、seed 数等信息；
- 将图表位置调整到“首次引用后紧随出现”的阅读流；
- 用 PGFPlots / booktabs / algorithmic 风格统一图、表、算法；
- 明确区分“已有数据支持的可视化”和“缺少原始曲线日志、不能额外补造的数据”。

这一天的重点是论文呈现质量、可复现性表达和审稿人可读性，而不是算法本身的新修改。

## 关键对话与决策

### 1. 确认论文主文件与编辑对象

对话中确认当前主要 LaTeX 源文件为：

- `nips2026.tex`

随后围绕该文件进行了图、表、算法和布局修改。今天的所有产出都应理解为对该论文主文件的 camera-ready 视觉与结构优化。

### 2. 坚持 Zero Hallucination 原则

一个非常重要的决策是：所有图表数字必须来自原论文已显式报告的数据，不能凭审美或趋势补造。

具体落实为：

- 对 [[FTC Metric]] / Reward 曲线：确认本地论文源没有 per-timestep seed logs，因此不能生成真实逐时间步置信区间；
- 若使用 CI band，只能基于已报告的 4-seed summary std 进行透明推导，且不能暗示存在未报告的逐步统计；
- 后续又进一步收紧：Figure 2 曲线数据改为“仅由原文已给坐标经过 5 点滑动平均得到”的版本，避免隐含插值或额外采样点争议；
- 新增 2×4 multi-task figure 时，对于未报告的 task-method pair，不补造数据，只显式保持空缺/不画。

这对 [[TC-PPO Algorithm]] 论文很关键：方法声称的稳定性改进需要依赖可信实验，而不是视觉上“更好看”的虚假平滑曲线。

### 3. 图表顺序按 NeurIPS 阅读流重排

今天做出的结构决策包括：

- Figure 1 移到 Policy Architecture / Sec. 3.6 附近；
- Algorithm 1 保持在 Optimization 相关段落之后；
- Table 1 放在 Main Results 之后；
- Figure 2 使用 `figure*`，与 Figure 3 一起放在 Training Dynamics 之后；
- Table 2 和 Figure 4 放在 Ablations 之后；
- 后续又在 Table 1 后插入新的 full-width 2×4 multi-task summary figure；
- 为新增主图补充正文引用，保证“先引用，再紧随放置”。

这个顺序更符合审稿人的阅读路径：先理解方法结构，再看优化流程，再看主结果、训练动态和消融。

### 4. 表格风格统一为 booktabs

表格方面做了显著规范化：

- Table 1 改为 grouped / widened `table*` 风格；
- Table 2 去掉剩余 vertical rule，避免违反 booktabs 风格；
- supplementary tables 也统一 caption、spacing、placement 风格；
- 表格 caption 强调实验设置、seed、budget 等上下文，减少读者误解。

这属于 camera-ready 质量提升，不改变实验结论。

### 5. 算法伪代码重写但不改变方法

Algorithm 1 被重写为 NeurIPS 常见的 `algorithm + algpseudocode` 风格：

- 增加显式输入/输出；
- 将稳定化逻辑拆成更清晰的逐行步骤；
- 注释对齐 Eqs. (6–8) 的记号；
- 保持 [[TC-PPO Algorithm]] 方法本身不变。

这个修改的价值在于让审稿人和复现者更容易理解 [[Temporal Consistency]] 约束如何嵌入 PPO 更新流程。

## 新洞察

### 1. 论文视觉质量必须服务于“可信性”，不是只服务于“好看”

今天最重要的写作/可视化洞察是：对于 RL 论文，漂亮的 learning curve 很容易造成数据幻觉风险。尤其在没有 per-timestep seed logs 的情况下，不能为了符合 HumanoidBench 风格而伪造逐步置信区间。

因此更稳妥的策略是：

- 坚持只画已报告坐标；
- 平滑方法需要可解释，例如明确为 5 点滑动平均；
- CI 来源必须说明；
- 未报告任务/方法组合宁可留空，也不补线。

这对 [[FTC Metric]] 的呈现尤其重要，因为 FTC 本身是稳定性指标，如果可视化过程不稳定/不透明，会削弱指标可信度。

### 2. [[Mode A-B-C Taxonomy]] 相关叙事需要图表顺序支撑

虽然今天主要做 LaTeX 可视化，但隐含推进了 [[Mode A-B-C Taxonomy]] 的论文叙事：  
图表顺序从“方法结构 → 优化流程 → 主结果 → 训练动态 → 消融”展开，更容易让读者理解为什么 [[TC-PPO Algorithm]] 能缓解并行 PPO 在 humanoid control 中的训练不稳定问题。

也就是说，taxonomy / failure mode 不是只靠文字解释，而需要通过主结果、training dynamics 和 ablation figure 逐层建立证据链。

### 3. Algorithm 1 是复现入口，不只是论文装饰

重写 Algorithm 1 的意义不只是排版美观。对于机器人学习论文，伪代码往往是复现者理解方法的第一入口。

今天将 Algorithm 1 对齐 Eqs. (6–8)，并加入 explicit I/O 和逐行稳定化逻辑，使其更像一个可执行算法说明，而不是概念性段落摘要。这有助于后续把论文方法与代码实现、实验配置和消融项对应起来。

### 4. 主结果图应承担“跨任务稳定性摘要”的功能

新增 full-width 2×4 multi-task figure 的决策说明：单个曲线或单张表不足以展现 humanoid control 多任务 setting 下的整体模式。  
一个按任务网格组织的主图更适合回答审稿人关心的问题：

- TC-PPO 是否只是某个任务有效？
- 是否在多个 locomotion / humanoid control setting 下有一致趋势？
- 是否有任务上的缺失数据？
- 与 baseline 的差异是否可视化清楚？

因此，这张图是论文主结果叙事的关键承载物。
