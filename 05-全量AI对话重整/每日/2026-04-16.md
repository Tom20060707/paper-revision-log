#### `powerlift hard gating`

| seed | reward |
|---|---:|
| seed0 | 72.0 |
| seed1 | 77.0 |
| seed2 | 65.0 |
| mean±std | **71.3±6.1** |

与已有 soft gating 对比：

| method | reward |
|---|---:|
| soft gating | **90.5±0.1** |
| hard gating | **71.3±6.1** |
| vanilla PPO | **99.7±8.3** |

关键结论：在 `powerlift` 这个 [[Mode A-B-C Taxonomy|Mode A]] 任务上，soft gating 相比 hard gating 不仅分数更高，而且方差显著更小。当天对话中将其总结为 soft 比 hard 高约 **+27%**，且方差从 **6.1** 降至 **0.1**。这是 soft gating 稳定性优势的核心证据。

### 3. 服务器资源决策

一开始两个 pending 任务分配到 Server B 与 Server C；随后用户指出其实一个服务器足够。经检查：

- MuJoCo + 小网络训练主要是 CPU-bound；
- GPU 利用率约 1%–3%；
- 两个任务合并到一台机器不会造成 GPU 瓶颈。

最终决策：

- 保留 **Server B（三卡 4090）** 同时跑：
  - `vanilla [512×2] slide seed2`，10M steps；
  - `powerlift soft seed0`，warmup 1M + TC-PPO 9M。
- 清空并关闭 **Server C（A800）**，节省资源。

### 4. 关于 MuJoCo、Isaac、sim-to-sim、sim-to-real 的战略判断

用户表达了对 MuJoCo 实验速度和 GPU 利用率的强烈不满，并提出是否换 Isaac 或做 simulation-to-simulation。

当天形成的判断：

- **当前这篇不建议换 Isaac**  
  因为现有论文已经积累了大量 MuJoCo/HumanoidBench 实验数据，换 Isaac 意味着任务定义、接触模型、reward scale、[[FTC Metric]] 分布、P5/P75 阈值全部重测，基本等于重做论文。

- **sim-to-sim 可以作为下一篇论文方向**  
  如果能证明 FTC 只依赖接触事件时间结构，因此在 MuJoCo→Isaac 或 Isaac→MuJoCo 迁移中比力/力矩指标更稳健，这可以发展成新的跨仿真器泛化论文。

- **当前论文不需要真实 sim-to-real 实验**  
  TC-PPO 是训练稳定性算法论文，不是部署论文。HumanoidBench 本身是仿真 benchmark。当前更合理的做法是在 Limitations 中诚实讨论 sim-to-real，并提供可迁移性的定性论证。

### 5. CoRL/TRO 现实要求的处理

当天查证并总结：

- TRO 官方不硬性要求真机实验，允许 theory/simulation/experiment 的组合。
- CoRL 鼓励 real-robot experiments **or** convincing evidence that simulation results are transferable。

因此论文中需要增强的是“为什么 TC-PPO/FTC 有可迁移性依据”，而不是立刻做真机实验。

## 论文/实验/代码/文档产物

### 1. 论文新增五类审稿人预防性回应

对 `main.tex` 加入了五类主动防御性论述，并成功编译：

1. **大并行环境下 TC-PPO 是否冗余**
   - 加入位置：Computational Overhead / Limitations 附近。
   - 核心表述：
     - TC-PPO 是 selective low-overhead safety mechanism，不是 universal performance booster；
     - [[Mode A-B-C Taxonomy|Mode C]] 任务中 gate 永久关闭，开销近似为零；
     - 在 N≥4096 且 vanilla PPO 已稳定时，TC-PPO 自动保持 inactive，与 vanilla PPO 行为一致；
     - 这是“zero-cost insurance”特性。

2. **P5/P75 是否只是调参**
   - 加强阈值校准段落。
   - P5 被解释为健康接触分布的异常低尾，保证 gate 只在真正不稳定时触发。
   - P75 被解释为可达性条件，保证恢复后的策略能关闭 gate，是 Mode A 的必要条件。
   - Δτ≈0.07 被表述为百分位语义结果，而非额外自由参数。
   - P45 窗口过窄导致 chattering，与 `tab:window_calibration` 对应。

3. **sim-to-real 传感器噪声**
   - 加入 FSR 噪声、迟滞、5–15ms 时间抖动可能增加 FTC 方差和 false-positive gating 的讨论。
   - 强调三个缓解机制：
     - sliding window `n=100` 提供统计平滑；
     - `max(0,·)` 截断与 ε 项避免病态行为；
     - soft gating 相比硬二值门控对瞬时尖峰更鲁棒。

4. **软门控权重物理意义**
   - 对 `w=[-2.0,-0.5,1.0]` 追加解释：
     - 来源于诊断日志中区分 collapse trajectory 与 stable trajectory 的 Mahalanobis 方向初始化；
     - FTC 的较大负权重反映其作为主要崩溃预测器；
     - contact entropy 的较小负权重降低对健康多肢接触多样性的惩罚；
     - contact interval 的正权重强调长时间无接触这一不稳定信号；
     - soft gating 是 hard gating 的严格泛化。

5. **FTC leading indicator 的因果边界**
   - 明确当前证据支持 FTC 是 Layer-2 failure 的 leading indicator，但不证明跨所有崩溃模式的普遍因果性。
   - 将因果 claim 控制在“充分但非必要”的范围内。

编译结果曾记录为：29 页，无错误、无 undefined references。

### 4. sim-to-real 可迁移性论证加入 Limitations

在 Section 7 的 `Real-robot deployment` 段末追加定性论证：

- [[FTC Metric]] 只依赖 contact timestamps，也就是 touch-down / lift-off 的二值事件序列；
- 不依赖 contact forces、joint torques、stiffness、damping 等物理引擎相关量；
- 因此 FTC 的分布形状相对不敏感于仿真器物理参数差异；
- P5/P75 百分位阈值有望跨仿真器或真实硬件只需少量 re-profiling，而非完全重标定；
- TC-PPO 的 gate signal 相比依赖力/力矩的方案，理论上更不容易出现 spurious activation。

注意：这部分没有加入任何无实验支撑的定量 sim-to-real claim，只作为 CoRL 审稿所需的 transferability argument。

## 新洞察

### 1. soft gating 的价值不仅是均值提升，更是方差压缩

`powerlift hard gating = 71.3±6.1`，而 `soft gating = 90.5±0.1`。这说明 soft gating 在 [[Mode A-B-C Taxonomy|Mode A]] 中的核心优势不是简单地“更强干预”，而是通过连续化门控避免硬切换导致的策略震荡与训练不稳定。

这对论文叙事很重要：soft gating 应被定位为 [[TC-PPO Algorithm]] 的稳定性机制，而不是一个可有可无的实现细节。

### 2. `[512×2] slide seed2` 可能不是异常，而是慢收敛轨道

原先多次补跑都得到约 19.85，曾怀疑是进程未写入新目录或异常值。进一步看 reward 曲线后，判断更可能是：

- seed2 在 5M 时仍在持续上涨；
- seed0/seed1 在 5M 已接近 29；
- seed2 进入了更慢的优化轨道。

因此应该延长到 10M，而不是简单删除异常值。这对 [[FTC Metric]]/架构 scaling 论证更诚实：如果 10M 仍低，则说明 `[512×2]` vanilla PPO 确实存在高方差。

### 3. MuJoCo GPU 利用率低是设置本身导致，不是服务器问题

当前实验只有少量环境，小网络 `[512×2]` / `[512×4]`，MuJoCo rollout 由 CPU 子进程主导，GPU 只是小规模前向/反向。因此 1%–3% GPU 利用率是预期，不应误判为资源分配错误。

如果要真正提升吞吐，只能：

- 增加并行环境数；
- 增大网络；
- 或迁移到 Isaac/GPU simulator。

但这些都会改变当前论文实验设置。

### 4. sim-to-real 不必做，但 transferability argument 必须写清楚

对于训练稳定性算法论文，真实机器人不是硬要求；但需要说明为什么仿真结论不是完全不可迁移。FTC 的优势在于只依赖接触事件时间戳，不依赖力、力矩、刚度、阻尼等引擎敏感量。这是当前论文最合理、成本最低、学术上也诚实的 sim-to-real 防御点。

### 5. 图表问题有时不是 LaTeX，而是图文件 bounding box

Figure 7 的“大空白”最初看起来像 IEEEtran float 问题，但真正根因是 matplotlib 标签超出坐标轴，`bbox_inches='tight'` 扩展了 PDF bounding box。  
以后图上方空白应先检查：

- 图像自身 bounding box；
- 轴外 annotation；
- y-axis clip；
- `bbox_inches='tight'`；
- 再检查 LaTeX float。
