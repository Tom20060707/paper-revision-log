今天的核心工作围绕 [[TC-PPO Algorithm]] 论文稿件压缩与重构展开，目标是把当前偏“技术报告/补充材料合集”的长稿，改成更适合 CoRL/NeurIPS 正文风格的论文主文。

1. 用 [[FTC Metric]] 描述足端接触时序一致性；
2. 用 FTC 触发 PPO 的自适应稳定化机制，即 [[TC-PPO Algorithm]]；
3. 用 H1 walking 主实验展示 reward、collapse depth、success rate 等核心结果。

今天形成了明确判断：当前稿件的问题不是“内容不够”，而是内容过多、主线被大量附录型材料稀释。后续写作应主动收缩论证范围，避免把 pilot、pending、reproducibility checklist、代码片段、完整数学补充都塞进主文。

## 关键对话与决策

### 1. 明确论文压缩目标

今天多次恢复上下文并继续同一个任务：将原长稿从约 31 页压缩为约 9 页主文，其余内容移入附录。对话中确认源文件是一篇关于 [[TC-PPO Algorithm]] 的 LaTeX 论文，包含大量实验、理论和可复现性补充。

形成的压缩策略：

- **Abstract**：保留 FTC、FTC-conditioned adaptive stabilization、H1 benchmark 结果三条核心贡献；弱化或删除长训练 pilot 的展开。
- **Introduction**：压缩到约 1 页，合并重复动机段落，保留贡献列表但减少铺陈。
- **Related Work**：压缩到约 0.5 页，只保留与 humanoid RL、PPO stability、contact temporal consistency 直接相关的 3–4 个核心段落。
- **Method**：压缩到约 2 页，只保留 [[FTC Metric]] 核心公式、[[TC-PPO Algorithm]] 目标函数和 Algorithm 1；形式化定义、引理、证明移到附录。
- **Experiments**：主文只保留 H1 walking 主结果、训练动态和必要 ablation；其余 robustness、cross-morphology、long-horizon pilot、sensitivity analysis 等移至附录。
- **Limitations / Conclusion**：保留，但压缩措辞，避免过度声明。

### 2. 明确“主文不该承载”的内容

今天对稿件结构做了较清晰的分类，决定以下内容不应作为主文主体：

- Full Implementation Details；

对 0.6B long-horizon pilot 的处理形成了明确决策：不能让它成为主证据链。

对话中出现过的可信数字包括：

- collapse near `2.1×10^8` steps；
- peak `5042.38`；
- endpoint `5001.4±58.8`；
- `5.94×` 提升；
- 该部分被标注为 protocol-only / pilot evidence，而非 matched-baseline superiority claim。

因此，今天决定：这类 0.6B 长训练结果可以放入附录或只在主文一句带过，不能作为主表或主论点支撑。原因是它本身已被定义为 pilot / protocol-only，不具备与 matched baseline 直接比较的主证据资格。

## 新洞察

### 1. 论文主线应是“稳定性信号驱动 PPO 自适应”，不是“堆实验规模”

今天最关键的写作洞察是：这篇论文真正有价值的地方，不是长训练曲线或大量补充实验，而是把足端接触时序一致性抽象为 [[FTC Metric]]，再把它接入 PPO 稳定化机制，形成 [[TC-PPO Algorithm]]。

因此主文应围绕一个问题展开：

> humanoid locomotion 中，策略崩溃是否可以通过接触时序一致性的早期信号来识别，并用该信号调节 PPO 更新，从而减少 catastrophic collapse？

这比展示许多 pending 或 pilot 结果更有论文说服力。

### 2. FTC 应作为“诊断 + 控制触发信号”，而不是单纯指标

今天的重构建议隐含了一个更强的定位：[[FTC Metric]] 不只是 evaluation metric，而是能进入控制/优化闭环的 temporal-consistency signal。

这使它区别于普通 reporting metric：

- 它能描述足端接触间隔的规律性；
- 它能提示 policy 是否进入不稳定 regime；
- 它能作为 TC-PPO 中 adaptive stabilization 的触发条件；
- 它连接了 locomotion 中的接触模式与 PPO 更新稳定性。

这个定位应在方法章节中强化。

### 3. pending 和 pilot 会削弱主张，而不是增强主张

今天明确了一个论文写作风险：为了显得“实验丰富”而加入 pending / pilot / protocol-only 内容，反而会让审稿人怀疑主张边界。

尤其是：

- Atlas / Digit pending baseline 不应进入主文；
- 0.6B pilot 不能替代 matched baseline；
- protocol-only evidence 不能支撑 superiority claim；
- 主文应优先展示 completed, matched, interpretable experiments。

这对于后续整理 [[Mode A-B-C Taxonomy]] 或稳定性模式分类也有启发：分类与论证必须基于可验证数据，而不是计划中的实验。

### 4. 附录不是垃圾桶，而是可信度支撑层

今天的删减不是简单“砍内容”，而是重新分层：

- 主文负责论点；
- 附录负责证明、复现、扩展和审稿补充；
- checklist 与 implementation details 不能抢占正文叙事；
- 实验协议可以完整保留，但不应打断主线。

这个结构更适合公开仓库中的论文版本，也更利于后续生成 arXiv / camera-ready / supplementary package。
