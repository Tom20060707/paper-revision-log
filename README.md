# 论文修改日志：接触域一致性与稳定性

欢迎来到这个有点“长出来”的论文仓库。

它不是一份冷冰冰的聊天记录，也不是把所有讨论原样倒进 GitHub。更准确地说，它记录了一篇人形机器人强化学习论文从模糊、摇摆、试错，到逐渐找到主线的过程。

一开始，我们以为自己在做一个 PPO 改进方法，名字叫 **TC-PPO**。后来越写越发现，真正有意思的点不只是“某个算法有没有赢”，而是另一个更像科研问题的问题：

> 人形机器人在接触密集任务里为什么会训练不稳？  
> 在 reward 明显掉下去之前，接触模式会不会已经先乱了？  
> 如果这个“先乱起来”的信号能被看见，PPO 能不能少崩几次？

所以，现在这篇论文更像是在讲：

> 接触一致性帮助我们提前看见退化；PPO 的稳定化机制尝试把训练拉回比较健康的接触状态。

这句话听起来比 “TC-PPO 提升了多少多少” 慢一点，但也更稳一点。科研里有时候就是这样：真正值得写清楚的，往往不是最响亮的口号，而是那个最能解释现象的角度。

## 这个项目现在的核心直觉

人形机器人训练难，不只是因为动作维度高。它还有一个很麻烦的地方：身体一直在和环境接触。

脚踩地、手推物体、身体保持平衡，这些接触一旦变乱，PPO 的训练就可能慢慢滑进坏状态。尴尬的是，reward 有时候反应得没那么早；等 reward 明显崩了，接触模式可能已经乱了一阵子。

这就是“接触一致性”想做的事：不要等最后成绩掉下来，先看看机器人和环境的接触方式是不是开始不靠谱了。

现在几个词可以这样理解：

- **接触一致性**：接触行为是不是还有规律、还可比较。
- **接触域稳定性**：策略训练和执行是不是能在复杂接触里站得住。
- **CSC**：更通用的接触面一致性思路，不只盯着脚。
- **CTS/FTC**：早期从足部或接触时序出发的版本，适合支撑类、准周期任务。
- **Mode A/B/C**：不是给任务贴好坏标签，而是在描述训练状态和方法边界。

## 为什么还保留旧名字

你会在仓库里看到 TC-PPO、FTC、CTS、CSC 这些不同名字。它们不是互相打架，而是记录了这篇论文一路摸索的过程。

大概是这样一路走过来的：

```text
TC-PPO / FTC
→ 用接触时序解释 PPO 失稳
→ CTS-PPO
→ 接触域一致性与稳定性
→ CSC 作为更宽的接触面一致性框架
```

最终写论文时，主线会更靠近“接触域一致性与稳定性”。旧名字不会被抹掉，因为它们说明了想法是怎么长出来的；但它们也不会再喧宾夺主。
