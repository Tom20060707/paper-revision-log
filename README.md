# 论文修改日志：接触域一致性与稳定性

这不是一篇“聊天记录合集”。这个仓库记录的是一篇人形机器人强化学习论文的想法怎么一步步变清楚。

最开始，我们想写的是一个 PPO 改进方法：**TC-PPO**。后来发现，真正有价值的不是“某个算法在所有任务上都赢”，而是一个更扎实的问题：

> 人形机器人为什么会在接触密集任务里训练崩掉？  
> 崩掉之前，接触模式有没有先变坏？  
> 如果能提前看见这种退化，PPO 能不能更稳一点？

现在的论文主线已经收束为：

> **接触域一致性与稳定性**：用接触一致性做崩溃前诊断，用 PPO 稳定机制抑制接触域退化。

## 现在到底讲什么

一句话：

> 高维人形 PPO 的难点，不只是动作空间大，而是身体接触模式会退化；接触一致性是能提前看到这种退化的信号，稳定化机制则负责把训练拉回可控区域。

更具体一点：

- **一致性**：接触行为有没有形成可重复、可比较的模式。
- **稳定性**：策略训练和执行会不会因为接触模式乱掉而崩溃。
- **CSC**：更通用的接触面一致性框架。
- **CTS/FTC**：CSC 在支撑接触或准周期接触上的早期实例。
- **Mode A/B/C**：不是任务好坏，而是接触稳定域的不同状态。

## 为什么保留旧词

仓库里会看到 TC-PPO、FTC、CTS、CSC 等不同名字。这不是整理错误，而是论文想法真实演化的痕迹。

大致过程是：

```text
TC-PPO / FTC
→ 接触丰富 PPO 的失稳诊断
→ CTS-PPO
→ 接触域一致性与稳定性
→ CSC 总框架 + CTS/FTC 特例
```

当前写论文时，要以 **接触域一致性与稳定性** 为主线。旧词可以保留在历史解释里，但不要让它们抢走主命题。
