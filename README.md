# 论文修改日志：接触域一致性与稳定性

这里放的是一篇人形机器人强化学习论文的修改记录。

不是“成果展示墙”，也不是把聊天记录一股脑倒出来。更像是一个研究生一路改论文时留下的工作台：有已经想清楚的部分，也有改过方向的痕迹；有公式、术语，也有一些当时绕弯路留下来的记录。

这篇论文最早是从 **TC-PPO** 开始的。当时的想法比较直接：PPO 在人形机器人接触任务里容易不稳，那能不能用接触时间的一致性帮它稳一点？

后来越看实验，越觉得问题不该只写成“我改了 PPO”。真正值得讲的是：

> 人形机器人训练出问题时，reward 往往不是第一个报警的东西。  
> 接触模式可能更早开始变乱。  
> 如果能提前看见这种变化，也许就能让 PPO 少往坏方向更新几步。

所以现在的主线大概是：

> 用接触一致性去观察训练是不是正在变坏，再用更谨慎的 PPO 更新把它往稳定状态拉回来。

说得再白一点：不是为了造一个很响的新名词，而是想把“接触为什么影响训练稳定性”这件事讲清楚。

## 现在的理解

人形机器人难，不只是因为动作维度高。它一直在和世界接触：脚踩地、身体保持平衡、手推物体、末端执行器插入或碰撞。

这些接触一旦变乱，训练就可能开始走歪。麻烦的是，reward 有时反应得比较慢；等 reward 掉下来，策略可能已经在坏接触模式里待了一阵。

所以这篇论文想看的是：接触模式有没有办法提前提醒我们，训练正在变糟？

几个词可以先这样理解：

- **接触一致性**：接触行为是不是还有基本秩序。
- **接触域稳定性**：策略在复杂接触里是不是还能稳住。
- **FTC / CTS**：早期从脚部接触、接触时序出发的想法。
- **CSC**：后面更宽一点的说法，不只看脚，也不强迫所有任务都有周期节奏。
- **Mode A/B/C**：不是任务排名，而是几种训练状态和方法边界。

## 为什么保留旧名字

仓库里还会看到 TC-PPO、FTC、CTS、CSC 这些名字。它们不是要同时当论文标题，而是这篇论文一路改出来的痕迹。

大概过程是：

```text
TC-PPO / FTC
→ 用接触时序解释 PPO 失稳
→ CTS-PPO
→ 接触域一致性与稳定性
→ CSC：更宽的接触面一致性思路
```

最后写论文时，会以“接触域一致性与稳定性”为主。旧名字保留下来，是为了让别人看得出这个想法不是凭空冒出来的。
