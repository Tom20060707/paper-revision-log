# 总览

## 当前一句话

这篇论文已经从“提出一个 PPO 变体”演化为：

> 研究人形机器人接触域中的一致性与稳定性，用接触一致性进行崩溃前诊断，用稳定化机制改善 PPO 在接触密集任务中的训练可靠性。

## 为什么会演化

早期的 TC-PPO/FTC 叙事有价值，但边界也很明显：

- 如果只说性能提升，Mode B/C 和负结果会削弱论文说服力。
- 如果只说 FTC/CTS，容易被理解成一个足部或周期步态指标。
- 如果把 `1 - CV²` 当成通用公式，非周期任务会直接反驳它。
- 如果把 kitchen、insert、truck 等任务都纳入同一个接触间隔指标，会显得过度泛化。

因此当前论文应把旧指标放进更大的框架：

```text
CSC 总框架
  └─ CTS/FTC：支撑接触或准周期接触的已验证实例
```

## 五个阶段

### 1. TC-PPO / FTC 起点

目标是解释 PPO 在高维人形接触任务中的训练不稳定。FTC 用于观察接触时序是否在 reward 崩溃前发生退化。

这一阶段贡献了最早的实验入口和 Mode A/B/C 分类。

### 2. 诊断框架

论文逐步从“性能提升”转为“诊断框架”：

- Mode A：接触退化可恢复，保护机制有效。
- Mode B：阈值或接触结构不匹配，保护状态可能锁死。
- Mode C：接触稳定，但任务可能仍无进展。

这使得负结果可以被解释，而不是被隐藏。

### 3. CTS-PPO

为了避免 FTC 过窄，论文转向 Contact Timing Stability：

```text
CTS = Contact Timing Stability
CTS-PPO = Contact Timing Stability-Guided PPO
```

这一步强调接触时序稳定性与高维控制灾难之间的关系。

### 4. CSC 总框架

进一步讨论后发现，单纯的接触间隔规律性不适合所有任务。许多任务是非周期、阶段化或事件驱动的。

因此提出更一般的：

```text
CSC = Contact-Surface Consistency
```

CSC 不强迫任务具有周期接触，而是比较当前接触事件分布与稳定参考分布。

### 5. 一致性与稳定性分工

最终写作口径：

| 概念 | 角色 | 论文中的作用 |
|---|---|---|
| 接触一致性 | 诊断信号 | 发现接触域退化 |
| 接触域稳定性 | 目标性质 | 衡量策略训练是否可靠 |
| CSC/CTS | 观测指标 | 崩溃前预警 |
| PPO 稳定机制 | 干预方法 | 抑制崩溃、恢复稳定域 |

## 写作原则

- 不把 TC-PPO 写成简单 performance booster。
- 不把 FTC/CTS 写成所有任务通用指标。
- 不把未验证的 hand-object、object-environment、tactile CTS 写成已有结果。
- 允许把操作任务和 kitchen 写成边界分析。
- 所有实验数字必须回到真实日志、脚本或论文源码。
