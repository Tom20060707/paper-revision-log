# 论文叙事主线

## 最终主线

TC-PPO 的强点不是在所有任务上提高 reward，而是提供一个接触驱动的失稳诊断框架。

## 推荐表达

1. PPO 在 contact-rich humanoid tasks 中会出现长时程不稳定。
2. FTC 捕捉接触间隔规律性，是崩溃路径的上游信号。
3. Mode A/B/C taxonomy 解释 TC-PPO 何时有效、何时误伤、何时无关。
4. Adaptive threshold calibration 和 soft gating 将方法从硬阈值启发式提升为可校准流程。
5. PBRS/Kitchen 用于展示 FTC 的边界，而不是无数据支持的新成功案例。

## 不推荐表达

- “TC-PPO universally improves HumanoidBench performance.”
- “PBRS solves kitchen-class tasks.”
- “Mode C is simply success.”
- “Negative Mode B results are failures to hide.”

## 负结果的用法

Mode B 任务应被主动解释：当门控抑制了任务必要更新，FTC 无法恢复，gate 无法关闭，形成负反馈锁死。这个诊断本身就是 FTC framework 的贡献。
