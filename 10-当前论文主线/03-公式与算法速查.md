# 公式与算法速查

这份文档只讲“现在论文里这些公式和机制各自干什么”。它不是最终 LaTeX 版本，真正写进论文前还要和 `main.tex` 对齐。

## 1. 接触事件

对每个接触面，先定义一个 0/1 接触事件：

```text
cᵢ(t) = 1：第 i 个接触面在时刻 t 发生接触
cᵢ(t) = 0：第 i 个接触面在时刻 t 没有接触
```

这里的接触面可以是：

- 左脚-地面；
- 右脚-地面；
- 手掌-物体；
- 膝盖-环境；
- 其他任务相关接触面。

关键点：接触面要由任务决定，不能默认所有任务都只看脚。

## 2. 早期 CTS/FTC 公式

早期支撑接触场景里，可以看相邻接触事件之间的时间间隔：

```text
Δtₖ = tₖ - tₖ₋₁
```

如果这些间隔很稳定，说明接触节奏比较规律。

常用的轻量分数可以写成：

```text
CV² = Var(Δt) / (Mean(Δt)² + ε)

CTS = clip(1 - CV², 0, 1)
```

直觉：

- `CV²` 越大，接触间隔越乱；
- `CTS` 越高，接触节奏越稳定；
- `ε` 是很小的数，防止除以 0。

边界：

这个公式适合准周期支撑接触，不适合作为所有任务的通用指标。

## 3. CSC 总框架

为了处理非周期任务，需要更一般的接触面一致性：

```text
CSC = 1 - D(P_current, P_ref)
```

含义：

- `P_current`：当前窗口里的接触事件分布；
- `P_ref`：稳定参考分布，可以来自 warm-up、成功 rollout 或任务先验；
- `D`：两个分布之间的距离；
- `CSC` 越高，说明当前接触模式越接近稳定参考模式。

它比 `1 - CV²` 更宽，因为它不要求任务有固定周期。

可比较的内容包括：

- 接触占用时间；
- 接触切换顺序；
- 接触面转移模式；
- 阶段条件下的接触分布；
- 支撑接触和操作接触的组合。

## 4. 百分位阈值

阈值不是另一个指标，它是判断线。

用 warm-up 阶段的分数分布来定：

```text
τ_low = P5(score_warmup)
τ_high = P75(score_warmup)
```

意思：

- `τ_low`：低于这个值，说明接触模式明显变坏；
- `τ_high`：高于这个值并稳定一段时间，说明可以退出保护；
- `P5` 和 `P75` 是 warm-up 分数里的第 5 和第 75 百分位。

这比固定全局阈值更合理，因为不同任务天然有不同接触模式。

## 5. 硬门控

硬门控就是开关：

```text
如果 score < τ_low：
    进入 Protect

如果 score ≥ τ_high 且持续 K 个窗口：
    退出 Protect
```

优点：简单清楚。  
缺点：边界附近容易抖动，也可能保护过强。

## 6. 软门控

软门控不是开关，而是强度旋钮：

```text
α = clip((τ_high - score) / (τ_high - τ_low), 0, 1)
```

直觉：

- `α = 0`：接触状态正常，不需要保护；
- `α = 1`：接触状态很差，强保护；
- 中间值：按严重程度调节。

它可以用来调 PPO 的更新强度，例如：

```text
β_eff = β_base × (1 - kβ α)
λ_eff = λ_base × (1 - kλ α)
```

意思：

- 接触越不稳定，PPO 更新越保守；
- 接触恢复后，更新逐步放开。

具体公式要以最终 `main.tex` 为准，这里是机制速查。

## 7. CUSUM 漂移检测

CUSUM 用来看趋势是不是在变坏，而不是只看某一个点。

先把分数标准化：

```text
z_t = (score_t - μ_warmup) / (σ_warmup + ε)
```

再累积坏趋势：

```text
S_t = max(0, S_{t-1} + κ - z_t)
```

如果：

```text
S_t > h
```

就触发漂移报警。

解释：

- `μ_warmup`：warm-up 平均分；
- `σ_warmup`：warm-up 标准差；
- `κ`：容忍度；
- `h`：报警线；
- `S_t`：坏趋势累积量。

## 8. 接触加权 GAE

GAE 是 PPO 里估计 advantage 的方法。

普通 TD 残差：

```text
δ_t = r_t + γ V(s_{t+1}) - V(s_t)
```

接触加权后：

```text
δ'_t = w_t δ_t
```

再用 `δ'_t` 递推 advantage。

直觉：

- 接触质量差的样本，梯度影响降低；
- 接触可靠的样本，保留更多训练信号；
- 这比最后才缩放 advantage 更干净。

## 9. PBRS / Push

PBRS 是势函数奖励塑形：

```text
r'_t = r_t + γ Φ(s_{t+1}) - Φ(s_t)
```

它的角色不是“解决 kitchen”，而是处理一种情况：

```text
接触已经稳定，但任务没有进展。
```

这时可以用 Push 机制推动探索或任务进度。

边界：

- 如果 true task reward 仍为 0，就不能说任务已解决；
- 只能说它暴露了稀疏奖励或零梯度屏障。

## 10. 总算法流程

可以把当前机制理解为：

```text
1. 收集 rollout
2. 计算任务相关接触事件
3. 得到 CTS/CSC 分数
4. 用 warm-up 百分位阈值判断接触域是否退化
5. 如果退化，进入 Protect
6. 如果稳定但无进展，考虑 Push
7. 用加权 GAE / 保守 PPO 更新降低坏样本影响
8. 继续监控是否恢复稳定域
```

最重要的是：

```text
诊断和干预分开。
```

CSC/CTS 负责看见问题，PPO 稳定机制负责处理问题。
