# 小白导读

## 先把问题说成人话

PPO 是一种常见的强化学习算法。它让机器人不断试错，然后根据奖励更新策略。

在人形机器人任务里，问题会变得很麻烦。机器人不是一个小车，它有脚、腿、手、膝盖、身体，会不断和地面、物体、墙面发生接触。

接触一乱，训练就容易崩。

这篇论文想抓住的就是这个现象：

> reward 真正崩掉之前，机器人的接触模式往往已经开始不对劲。

如果能提前看见这个“不对劲”，就有机会让 PPO 少走弯路。

## 什么叫接触一致性

举个最简单的例子：上楼梯。

一个比较稳定的策略，脚接触台阶的节奏应该有一定规律。不是每一步都完全一样，但至少不会一会儿乱踩、一会儿拖脚、一会儿双脚悬空。

这种“接触行为有没有形成可重复模式”，就是接触一致性想看的东西。

它不是说动作要一模一样，而是说：

```text
该接触的时候大致接触；
该离开的时候大致离开；
不同 rollout 里的接触模式不要乱到完全没法比较。
```

## 什么叫接触域稳定性

接触一致性是信号，接触域稳定性是结果。

如果接触一致性开始下降，说明机器人可能正在进入坏状态。  
如果 PPO 的更新机制能把它拉回来，训练就更稳定。

所以论文不是为了追求“接触越一致越好”，而是想用它做预警。

## 为什么旧公式不够

早期我们用过一个很简单的想法：

```text
接触间隔越规律，分数越高。
```

这对走路、上楼、支撑这类任务有道理。

但很多任务不是这样：

- kitchen 是阶段化操作；
- insert 是接触精度问题；
- push/truck 是持续推动和物体交互；
- 短跑有起跑、加速、冲刺，不是稳定周期；
- 上楼梯走到平台时节奏也会变。

所以不能把一个周期步态公式包装成所有任务通用指标。

## 现在的解决方式

现在更合理的说法是：

> 不同任务有不同的接触面集合。我们要看这些任务相关接触面是否形成稳定模式。

这就是 CSC：Contact-Surface Consistency，接触面一致性。

CTS/FTC 仍然有用，但它们是 CSC 在支撑接触、准周期接触上的特例。

## Mode A/B/C 怎么理解

不要把 Mode A/B/C 理解成任务分类。它们更像训练状态。

```text
Mode A：发现要崩了，保护一下，后来恢复了。
Mode B：发现问题后一直恢复不了，可能阈值或接触面定义不对。
Mode C：接触看起来没崩，但任务也不一定成功。
```

这就是为什么 kitchen 不能简单说“接触稳定所以任务成功”。它可能接触稳定，但没有完成真正的操作目标。

## 这篇论文最该讲的故事

最清楚的故事线是：

1. 人形 PPO 容易因为接触域退化而失稳。
2. 接触一致性可以比 reward 更早暴露退化。
3. 早期 CTS/FTC 指标能解释一部分支撑接触任务。
4. 非周期和操作任务暴露了旧指标边界。
5. 因此论文升级到 CSC 总框架。
6. PPO 稳定机制利用这个诊断信号，抑制接触域退化。

这比“我提出一个 PPO 变体并在一些任务上提升”更稳，也更像一篇有思想的机器人学习论文。
