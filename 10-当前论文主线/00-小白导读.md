# 小白导读

这篇导读是给第一次接触这个项目的人看的。你不需要一上来就懂 PPO、GAE、CUSUM，也不用先背 TC-PPO、FTC、CTS、CSC 这些缩写。

我们先从一个很朴素的问题开始：机器人训练为什么会崩？

## 先把画面想出来

PPO 是一种常见的强化学习算法。你可以先把它理解成：机器人不断尝试动作，环境给它奖励，然后它根据这些经验调整下一轮策略。

如果是一个简单小车，事情还算清楚：往前开、别撞墙、到达目标。

但人形机器人麻烦多了。它有脚、膝盖、手、身体，还要和地面、物体、台阶、墙面发生接触。它不是单纯“移动一下”，而是在不断处理身体和环境之间的关系。

所以训练崩掉时，往往不是突然某一步“坏了”。更常见的是：接触模式先慢慢变奇怪，姿态开始不稳，动作开始绕远，最后 reward 才明显掉下来。

这篇论文想抓的，就是这个更早出现的“不太对劲”。

## 接触一致性到底是什么

先拿上楼梯举例。

一个学得还不错的策略，脚踩台阶的方式通常会有点规律。不是说每一步都必须像复制粘贴，但至少应该大致知道什么时候踩、什么时候离开、左右脚怎么交替。

如果一个策略开始乱踩、拖脚、悬空、落点飘来飘去，那我们大概会觉得：它还没完全失败，但已经不太妙了。

这个“不太妙”，就是接触一致性想提前捕捉的东西。

更白话一点：

```text
该接触的时候，别总是错过；
该离开的时候，别一直拖着；
不同尝试之间，接触模式别乱到完全没法比较。
```

它不是要求机器人动作僵硬一致，而是希望接触行为还有基本的秩序。

## 接触域稳定性又是什么

接触一致性更像一个观察信号，接触域稳定性才是我们真正想改善的东西。

如果接触一致性下降，它可能在提醒我们：训练正在进入一个坏接触区域。  
如果 PPO 的更新机制能根据这个提醒变得更谨慎，训练就有机会被拉回来。

所以这里的逻辑不是：

```text
接触越一致越好
```

而是：

```text
接触突然乱起来时，要早点发现；
发现以后，训练更新不要继续把策略推坏。
```

这点很关键。否则论文很容易被误读成“我们追求接触节奏整齐”，但其实我们更关心的是训练稳不稳。

## 为什么早期公式不够

早期有一个很直观的想法：

```text
接触间隔越规律，分数越高。
```

这个想法在走路、上楼、支撑这类任务上挺合理。因为这些任务确实有一点节奏感。

但问题来了：很多任务没有稳定节奏。

比如：

- kitchen 是一段一段的操作；
- insert 更关心接触精度；
- push/truck 是持续推物体；
- 短跑有起跑、加速、冲刺；
- 上楼梯到了平台，节奏也会变。

所以，早期的接触时间公式不能装成所有任务都适用的大理论。它有价值，但要放在合适的位置。

## 现在为什么转向 CSC

更宽一点的想法是 CSC：Contact-Surface Consistency，接触面一致性。

它不再只问“接触间隔是不是均匀”，而是问：

> 对这个任务真正重要的接触面，当前表现是不是还接近一个稳定、可解释的接触模式？

这就温和多了，也更像科研上能站得住的说法。

对上楼梯，重要接触面可能主要是脚和台阶。  
对推物体，重要接触面可能是手、身体和物体之间的接触。  
对插入任务，接触精度和阶段变化可能更重要。

所以 CTS/FTC 不是被否定了。它们仍然是有用的，只是更适合作为 CSC 在支撑接触、准周期接触里的一个具体版本。

## Mode A/B/C 怎么看

这里也先别把 Mode A/B/C 当成“任务等级”。它们更像是在描述训练状态。

```text
Mode A：发现接触要变坏了，保护机制介入，然后真的拉回来了。
Mode B：发现问题了，但没拉回来，可能接触面、阈值或任务结构没对上。
Mode C：接触看起来还行，但任务未必真的有进展。
```

Mode C 特别容易误解。它不是说“一切正常”，而是提醒我们：接触稳定和任务成功不是一回事。

Kitchen 就是很好的例子。机器人可能接触挺稳定，但如果没有打开柜门、移动物体、完成阶段目标，那任务还是没完成。

## 最后把故事串起来

这篇论文现在比较顺的故事是：

1. 人形 PPO 在复杂接触任务里容易失稳。
2. reward 崩掉前，接触模式可能已经先退化。
3. 接触一致性可以作为提前诊断信号。
4. 早期 CTS/FTC 能解释一部分支撑接触任务。
5. 非周期操作任务提醒我们，单纯看接触间隔不够。
6. 因此论文转向更宽的 CSC 框架。
7. PPO 稳定机制利用这些接触信号，让训练不要轻易滑进坏状态。

这样讲，论文就不只是“我改了一个 PPO 小模块”，而是在认真回答一个机器人学习里的现象：接触模式为什么会影响训练稳定性，以及我们能不能早点看见它。
