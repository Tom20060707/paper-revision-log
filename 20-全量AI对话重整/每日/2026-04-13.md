### 2. Mode A 证据链显著增强

最新实验显示，[[Mode A-B-C Taxonomy]] 中的 Mode A 不再只依赖 Stair：

- Stair：3/3 seeds，Mode A
- sit-hard：seed0/seed1 已确认 Mode A，seed2 当时仍在跑
- slide：seed0/seed1 已出现 Mode A，seed2 被安排补跑
- powerlift：后来在论文一致性检查中确认应归为 Mode A（adaptive soft gating）

这直接缓解了此前的 “P0 killer weakness”：如果 Mode A 只能在 Stair 上出现，论文主张会显得过窄；现在至少有 Stair、sit-hard、powerlift 三个任务支撑。

### 3. basketball / insert_normal 被明确归为 adaptive 阈值下的 Mode B

今天确认：

- basketball 三个 seed 全部是 Mode B
- insert_normal 三个 seed 全部是 Mode B
- 原因不是固定阈值实验中的 truck 式 Mode B，而是 adaptive P5/P75 把 `τ_low` 设到了任务训练期 FTC ceiling 上方，导致 gate 无法关闭

关键解释：

- basketball / insert_normal 的 `τ_low ≈ 0.49–0.51`
- final FTC 约在 `0.46–0.48`
- 因此 FTC 一直低于 `τ_low`，gate 持续触发，形成 Mode B

这成为一个新洞察：[[FTC Metric]] 的 adaptive calibration 并不总是安全，warmup FTC 高于训练期 FTC 时会产生“阈值上锁”问题。

### 4. vanilla baseline 的优先级被重新排序

论文当时最缺的不是继续盲目扩展 TC-PPO，而是给已经有 TC-PPO 结果的任务补 vanilla baseline：

这个决策的核心是：先把论文表格能填完整，而不是继续堆没有 baseline 的 TC-PPO 数字。

## 新洞察

### 1. Mode B 至少有两种机制，不能混写

今天明确区分了两类 Mode B：

- 固定阈值 Mode B：truck only
- adaptive P5/P75 Mode B：basketball、insert_normal

这两类现象不能混淆。truck 是固定全局阈值下 gate lock；basketball / insert_normal 是 warmup calibration 把 `τ_low` 设得过高，超过任务训练期 FTC ceiling。

这对 [[Mode A-B-C Taxonomy]] 很重要：Mode 名称相同，但成因不同，论文需要解释清楚。

### 2. adaptive P5/P75 的风险比预期更大

P5/P75 calibration 原本用于避免手工阈值，但今天的 basketball / insert_normal 结果说明：

> 如果 warmup 期 FTC 高于训练期稳定 FTC，adaptive `τ_low` 会被错误地设在任务可达上限之上，导致 gate 永远无法关闭。

这不是小 bug，而是一个机制性风险。它可以转化为论文贡献之一：TC-PPO 不仅提出方法，也通过 Mode B 暴露了接触一致性阈值设计的边界条件。

### 3. insert_small 的“坏结果”反而能加强阈值设计论证

vanilla insert_small 从旧值约 `26.0` 更新到 `32.4±2.3` 后，TC-PPO soft `23.1±0.9` 明显更差。短期看削弱 performance claim，但长期看有助于方法分析：

- 说明 FTC gate 不是万能提升器
- 说明错误 threshold 会造成过度保守
- 支持论文中 Mode B / failure mode 的系统分类

这比硬说“略低于 vanilla”更诚实、更有说服力。

### 4. 图表错误往往来自“配置语义混淆”

powerlift 同时存在：

- fixed Config B：Mode C，`68.0±6.9`
- adaptive soft gating：Mode A，`90.5±0.1`

如果图表不标清配置，就容易出现“同一任务既是 Mode C 又是 Mode A”的表面矛盾。后续所有表格都必须明确：

- 固定阈值还是 adaptive P5/P75
- hard gate 还是 soft gate
- 多少 seeds
- 是否 preliminary

### 5. 引用幻觉不一定是“完全虚构”，更常见是“错配”

今天的 citation audit 显示，最危险的错误不是看起来明显假的引用，而是：

- DOI 指向真实文章，但不是本文所述文章
- 作者、标题像真的，但页码 / 卷期 / 期刊错误
- 正文描述的数据是真实的，但 BibTeX 指向另一个不相关论文

这类错误审稿人一查就会严重损害可信度。后续论文写作必须把 bibliography 当作实验数据一样审计。

### 6. 生物力学内容应该降级为动机，而不是主线

用户指出论文中 biomechanics 内容可能过重，这是合理的。[[FTC Metric]] 的跨学科来源很重要，但它应服务于机器人学习贡献，而不是让论文变成 gait variability survey。

后续重写策略应是：

- 摘要 / Introduction 保留一两句跨学科动机
- 方法部分保留 FTC 与 CV² 的必要定义
- 删除过长的生物力学文献展开
- 把重点还给 TC-PPO、adaptive trust-region、Mode A/B/C 和实验结果
