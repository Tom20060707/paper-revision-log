# 2026-04-09 研究日记

## 今日主线

今天主要推进了 TC-PPO 相关论文从 CoRL 2026 投稿格式迁移到 IEEE Transactions on Robot Learning（T-RL）/ IEEE Transactions 模板的工作。重点不是改科学内容，而是将已有论文源文件转换为 IEEE 双栏 Transactions 格式，并解决迁移后出现的作者栏、盲审、代码块、表格、图注和公式溢出问题。

核心成果是：论文已成功使用 `IEEEtran` journal 模式编译，生成 22 页 PDF；作者栏已切换为 T-RL 双盲审稿所需的匿名版本；代码块、表格、图注、公式等明显遮挡或严重 overfull 问题已逐项修复。

## 关键对话与决策

1. **决定从 CoRL 模板切换到 IEEE Transactions/T-RL 模板**
   - 原始论文之前面向 CoRL 2026 准备，现在需要改为 IEEE Transactions on Robot Learning 投稿格式。
   - 明确约束：只改格式与模板结构，不改变科学内容。
   - 保留文本、公式、图表、算法、引用、caption 等内容不变。

2. **采用 IEEEtran journal 双栏格式**
   - 将原来的 CoRL/conference 风格模板切换为 IEEE Transactions 风格。
   - 使用 `\documentclass[lettersize,journal]{IEEEtran}`。
   - 保留与内容相关的 LaTeX 包，移除 CoRL 专属模板依赖。
   - 增加 `stfloats`，并将 `hyperref` 调整为 `hidelinks`，更接近 IEEE 投稿习惯。

3. **作者栏从完整信息转为盲审匿名版本**
   - 用户指出作者栏信息不全后，进一步确认 T-RL 投稿采用双盲审稿。
   - 因此当前投稿版不应暴露作者姓名、机构、邮箱。
   - 最终决策：盲审版显示 `Anonymous Author(s)`，隐藏全部作者与机构信息。
   - 用户提供了 ORCID：`0009-0006-0801-6180`。该信息被写入 camera-ready 版本的注释中，供录用后恢复完整作者栏时使用。
   - 公开日志中仅记录 ORCID 已用于 camera-ready 注释准备，不暴露其它隐私信息。

4. **解决 IEEE 双栏下代码块遮挡正文的问题**
   - 发现根因：原来的 `verbatim` 环境在 IEEE 双栏格式下不自动换行，长行会越过栏宽并覆盖相邻文字。
   - 决策：改用 `listings` 包，并设置 `breaklines=true`。
   - 所有 `verbatim` 块替换为 `lstlisting`，代码内容保持不变。

5. **检查并修复图片、表格、公式等排版溢出**
   - 用户进一步要求检查图片是否也存在遮挡或溢出。
   - 检查结果显示：图片本身基本没有明显溢出，主要问题来自表格、图注和公式。
   - 逐项修复了严重 Overfull：
     - `eq:coefs` 两个 `cases` 并排导致 113pt 溢出。
     - 软门控边界相关 `align*` 公式导致 80pt 溢出。
     - CV² 表格单栏过宽导致 88pt 溢出。
     - FTC trajectory 图 caption 有 27pt 溢出。
     - 附录 locomotion bar chart caption 有 34pt 溢出。

  - 仍存在极小 overfull/underfull 警告，属于双栏排版常见现象，待最终人工检查。

### 代码块排版修复

为避免代码块遮挡正文，完成：

- 新增 `listings` 包。
- 设置自动换行：
  - `breaklines=true`
- 将所有 `verbatim` 环境替换为 `lstlisting`。
- 代码内容不变，仅改变 LaTeX 展示环境。

这一步对 IEEE 双栏格式非常关键，因为 TC-PPO 论文中若包含配置片段、伪代码或日志样式文本，长行在单栏宽度下极易溢出。

### 表格、图注与公式修复

已完成的主要排版修复如下：

| 问题位置/对象 | 原问题 | 修复方式 |
|---|---|---|
| `eq:coefs` | 两个 `cases` 并排，在双栏中产生 113pt overfull | 改为 `align`，上下排列 |
| 软门控边界公式 | `align*` 中单行过长，产生 80pt overfull | 拆分长公式行 |
| CV² 表格 | `table[h]` 单栏内含 `p{5.8cm}` 宽列，产生 88pt overfull | 改为 `table*` 跨双栏，并将列宽扩至 `p{9.5cm}` |
| FTC trajectory 图注 | caption 中不可断行文本导致 27pt overfull | 改为 `figure*`，让 caption 使用双栏宽度 |
| 附录 locomotion bar chart | caption 产生 34pt overfull | 改为 `figure*` |
| 代码块 | `verbatim` 长行不折行，覆盖正文 | 改用 `lstlisting` + `breaklines=true` |

剩余警告：
- 约 5pt 和 0.6pt 的轻微 overfull。
- 目前判断为投稿排版可接受范围，但最终提交前仍应人工通读 PDF。

## 新洞察

1. **CoRL 到 IEEE Transactions 的迁移不只是换模板**
   - 单栏/会议风格切换到 IEEE 双栏后，最容易出问题的不是正文段落，而是：
     - 长公式；
     - 宽表格；
     - 代码块；
     - caption 中含路径、任务名、变量名等不可断行字符串。
   - 对 [[TC-PPO Algorithm]] 这类包含公式、实验配置、任务名和 ablation 图表的论文，迁移时必须系统检查 overfull log，而不能只看是否编译成功。

2. **双盲审稿优先级高于作者信息完整性**
   - 用户最初关注“作者栏信息不全”，但根据 T-RL 双盲要求，当前投稿版应匿名。
   - 因此正确策略不是补齐作者栏，而是：
     - 投稿版匿名；
     - camera-ready 注释中准备完整作者信息；
     - 录用后再恢复姓名、机构、邮箱、ORCID 等。

3. **代码块在 IEEE 双栏中要避免使用裸 `verbatim`**
   - `verbatim` 不自动换行，会直接突破栏宽。
   - 对论文中的配置片段、环境说明、伪代码补充材料，`listings` 更安全。
   - 这也提示后续写 [[FTC Metric]] 或 [[Mode A-B-C Taxonomy]] 相关补充时，若有命令行/配置片段，应默认使用可折行环境。

4. **图片本身不一定是问题，caption 反而更危险**
   - 检查中发现图片多使用 `\linewidth`、`\columnwidth` 或 `figure*`，本体未明显溢出。
   - 真正导致 overfull 的是 caption 中的长任务名、变量名或斜杠连接字符串。
   - 对机器人学习实验图尤其要注意任务名组合，例如 locomotion、trajectory、stair、bookshelf 等长标签。

5. **严重 overfull 应按类型处理，而不是盲目缩字体**
   - 今天没有简单通过缩小字体解决所有问题，而是分别处理：
     - 公式：拆行；
     - 表格：跨双栏；
     - 图注：扩大 caption 宽度；
     - 代码：启用自动换行。
   - 这种方式更符合 IEEE 论文质量，也更不容易破坏科学内容表达。

## 待办与风险

### 明日待办

1. **人工通读 `main.pdf`**
   - 尤其检查：
     - 首页标题与匿名作者显示；
     - abstract/keywords；
     - 所有跨双栏图表的位置；
     - 公式换行后语义是否仍清晰；
     - appendix 中图表是否被放置到合理位置。
   - LaTeX 编译无错误不等于视觉排版完全无问题。

2. **核对 T-RL 官方投稿要求**
   - 再次确认：
     - 当前是否应提交匿名 PDF；
     - 是否需要单独上传 source；
     - 是否有 page limit、supplementary material、graphical abstract 或 data availability 要求；
     - 是否允许/要求 ORCID 在投稿系统填写而不是 PDF 中显示。
   - 今日已按双盲逻辑处理，但最终以投稿系统要求为准。

3. **检查参考文献格式**
   - 已添加 `IEEEtranN`，但仍需确认：
     - 引文样式是否与正文命令兼容；
     - bibliography 是否存在缺失字段；
     - 是否有 CoRL/NeurIPS 风格残留。
   - 若 `IEEEtranN.bst` 在其它机器不可用，需要准备本地 `.bst` 文件或说明。

4. **检查内容是否被格式修改误伤**
   - 今日目标是“内容零修改”，但公式换行、表格跨栏、caption 调整可能影响可读性。
   - 需要 diff 检查：
     - 是否仅改变 LaTeX 结构；
     - 是否没有改动实验结论、数字、caption 含义、引用关系。

5. **为 camera-ready 版本保留切换方案**
   - 当前作者栏是匿名版。
   - 录用后需要恢复：
     - 作者姓名；
     - 机构；
     - 邮箱；
     - ORCID；
     - manuscript received/revised 信息。
   - 今日已在注释中准备 ORCID，但完整作者信息仍待最终确认。

### 风险

- **T-RL 作为新期刊可能有不同于标准 IEEE Transactions 的投稿细则**：当前采用标准 IEEEtran journal 方案，仍需以官方页面为最终依据。
- **跨双栏图表可能改变浮动位置**：`figure*`/`table*` 可能移动到页面顶部或后续页面，需要人工检查叙述顺序。
- **轻微 overfull 仍存在**：虽然严重溢出已清零，但最终提交前最好查看 log 和 PDF。
- **匿名信息泄露风险**：需检查 PDF metadata、致谢、补充材料、文件名或注释中是否泄露作者身份。
