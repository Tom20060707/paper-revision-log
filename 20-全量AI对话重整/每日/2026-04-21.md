## 新洞察

### 1. TC-PPO 的论文叙事应围绕“因果链”而非“性能提升”

今天明确了一个重要写作判断：[[TC-PPO Algorithm]] 的核心说服力不应只是“某些任务 return 更高”，而应是完整解释：

> 接触时序不规则是上游扰动；  
> [[FTC Metric]] 是这个上游扰动的在线可观测代理；  
> 优势函数方差与 trust-region violation 是中间机制；  
> 策略崩溃是下游结果；  
> TC-PPO 的门控正则化是在上游信号恶化时提前介入。

这能解释为什么 reward 不是好的早期信号：reward 已经是下游后果，等 reward 掉下来时可能已经进入不可逆崩溃区间。

### 2. 静态正则化失败的关键不是“强度不够”，而是“时机不对”

今天将静态正则失败重写为一个时变系统问题：

- 稳定阶段：
  - 接触较规律；
  - advantage variance 较低；
  - 过强正则会拖慢学习。

- 不稳定阶段：
  - 接触时序变得不规则；
  - `Var(Â_t)` 上升；
  - 原本足够的固定约束突然不够。

因此一个全局固定熵/KL/clip 设置无法同时适应两种状态。TC-PPO 的价值在于利用 FTC 判断“什么时候收紧”。

### 3. CUSUM 与 FTC-GAE 是两个正交控制旋钮

今天形成的算法结构很清晰：

- CUSUM：
  - 累计 FTC 低迷；
  - 降低单步噪声误触发；
  - 解决 intervention timing。

- FTC-weighted GAE：
  - 对低 FTC 时刻的 advantage 贡献降权；
  - 直接降低污染梯度进入 policy update 的强度；
  - 解决 update weighting。

这使 Enhanced TC-PPO 不只是“再加一个阈值”，而是从检测层与更新层分别控制崩溃路径。

### 4. [[Mode A-B-C Taxonomy]] 需要解释“失败模式”而不是只分类

Mode B 今天被重新理解为一个闭环问题：

> gate 开启 → 更新被压制 → 策略难以恢复规律接触 → FTC 继续低 → gate 无法关闭

这说明 Mode B 不是简单“任务太难”，而可能是门控机制与结构性低 FTC 任务之间形成了自维持保守循环。后续实验应关注如何区分：
- 真实接触不稳定；
- 任务天然低 FTC；
- 门控策略造成的恢复阻滞。

### 5. 扰动实验是论文可信度的关键补强

原论文主要在 HumanoidBench 任务自然训练过程中观察崩溃与恢复。今天的扰动实验规划使论文多了一种更主动的验证方式：

- 人为制造外部扰动；
- 控制扰动强度；
- 检查 FTC 是否先于 reward 恶化；
- 检查 TC-PPO / Enhanced TC-PPO 是否能在扰动下减少崩溃或方差。

这能把“观察到的相关性”推进为更强的机制验证。
