# 2026-02-23 研究日志：TC-PPO 论文审稿意见落地与证据对齐改稿

## 今日主线

今天的主要推进是围绕 `nips2026 (2) (1)_revised_v3.tex`，根据一份模拟 NeurIPS 2026 审稿意见，对 [[TC-PPO Algorithm]] 论文进行了较系统的“去幻觉 / 证据对齐 / 可复现性补强”改稿。

审稿意见给出的总体判断是 **Overall Score: 5/6，Weak Accept**，但条件是补齐若干关键缺口，尤其包括：

- [[FTC Metric]] 的形态泛化表述不能硬编码 H1 的 6 个接触相关肢体。
- Eq.7 与 Algorithm 1 中“软惩罚 + 硬 KL 约束”的机制需要一致。
- Algorithm 1 需要更明确地描述迟滞状态机，而不是只给出模糊触发逻辑。
- Vanilla PPO、High parallelism/no intervention 等基线定义需要更精确。
- 对 TD-MPC2、敏感性分析、统计显著性、图示证据的表述需要降级到已有证据能支持的范围。
- 图表和附录需要避免把示意图、复现实验或未完成 ablation 说成强结论。

今天完成的核心不是新增实验，而是把论文从“有些地方表述过强”调整为“方法、算法、实验叙述与证据链一致”，使其更适合进入 camera-ready 前的审稿应对阶段。

## 关键对话与决策

### 1. 决定优先修复“方法学硬伤”，而不是先润色文字

对话一开始明确了处理顺序：先扫描全文结构与关键词位置，再建立必须修改项清单。实际优先级被定为：

1. FTC 定义与形态泛化；
2. Eq.7 与 PPO 目标函数写法；
3. Algorithm 1 的迟滞状态机；
4. 基线定义与实验叙述；
5. 图表和统计措辞；
6. 编译检查。

这个决策是正确的，因为审稿意见中真正影响接收风险的是方法定义不一致和证据不足，而不是语言风格。

### 2. 将 `N_l = 6` 改为形态可配置的接触集合

原稿中存在类似“`N_l = 6`，H1 有 2 feet、2 hands、2 knees”的硬编码表述。审稿意见指出这会削弱方法的泛化主张：如果 FTC 必须为每种机器人手动指定接触相关肢体，那么它不是天然 morphology-agnostic。

今天的修改决策是：

- 不再把 `N_l = 6` 写成通用定义；
- 改为使用形态可配置的接触集合 `\mathcal{C}`，以 `|\mathcal{C}|` 表示接触相关部位数量；
- H1 只是一个特例，此时 `|\mathcal{C}| = 6`。

这使 [[FTC Metric]] 的定义更稳妥：它仍然需要任务/形态层面的接触集合配置，但不再把 H1 的结构伪装成通用算法常数。

### 3. 明确线性测量模型只是局部近似

审稿意见指出，离散接触检测不应被过度解释为严格线性观测。今天的处理是将相关数学表述降级：

- 明确线性测量模型是局部近似；
- 不再把接触事件或离散接触检测强行叙述为线性系统测量；
- 在 assumption 相关段落增加“未完成实证验证”的声明。

这个修改的意义是降低理论 claim 风险：论文仍可以使用局部近似来解释 FTC 触发机制，但不把近似解释包装成已证明的系统性质。

### 4. 统一 PPO 目标、软惩罚与硬 KL 约束

今天对方法部分做了较核心的一致性重构：

- 将 PPO 目标显式写为 `\mathcal{L}_{PPO}(\theta;\theta_{old})`；
- 新增或改写为“硬 KL 约束 + 软惩罚”的统一形式；
- 避免 Eq.7、正文描述、Algorithm 1 三者对 TC-PPO 更新机制的解释不一致。

这对 [[TC-PPO Algorithm]] 很关键，因为审稿人关注的是：FTC 触发之后到底是改变 trust region、增加惩罚、限制 KL，还是几者混用。今天的版本把机制表述收敛为一致框架。

### 5. Algorithm 1 重写为显式迟滞状态机

原 Algorithm 1 被指出状态机不完整。今天将其改为包含：

- `stabilization_active`
- `\Delta_{check}`
- `W`

的显式迟滞状态机。

这一步让 [[Mode A-B-C Taxonomy]] 或类似稳定模式切换逻辑更容易被读者复现：触发不是每一步无记忆判断，而是有检查间隔、窗口和激活状态的机制。对稳定控制论文来说，这比简单阈值触发更可信。

### 6. 决定对未完成实验不做强结论

今天多处选择“降级表述”，而不是补写没有数据支撑的 claim。例如：

- 敏感性分析不再宣称强显著性，只写“最高均值”；
- 表注中明确 pairwise 检验暂不宣称；
- TD-MPC2 相关表述限定在“本复现实验设定下”；
- 对未完成 specificity controls / ablations 不做强结论。

这是今天最重要的写作伦理决策：论文接受概率不应靠扩张 claim，而应靠让已有结果更可审计。

## 新洞察

### 1. FTC 的泛化性不是“无需配置”，而是“接触集合接口可配置”

今天最重要的概念澄清是：[[FTC Metric]] 不应被表述为完全不依赖形态知识。更准确的说法是，它通过接触集合 `\mathcal{C}` 提供一个形态可配置接口。

这一区别很关键：

- 过强说法：FTC 自动适用于任意形态，无需额外指定。
- 更可信说法：FTC 的计算形式对形态数量不固定，但需要为具体机器人定义接触相关集合。

这样既保留了方法的可迁移性，也避免了被审稿人抓住“硬编码 H1 limbs”的问题。

### 2. TC-PPO 的创新点需要从“又一个 adaptive PPO”中区分出来

审稿意见提醒：adaptive PPO / trust-region variants 已经很多，TC-PPO 的 novelty 不能只说“自适应调整 KL 或 clip”。今天的改稿方向隐含了一个更清楚的定位：

- TC-PPO 的触发信号不是 reward drop 或 generic KL drift；
- 而是来自接触稳定性/触觉一致性相关的 FTC；
- 它将接触模式失稳作为 trust-region 调整的依据；
- 迟滞状态机则减少频繁开关带来的训练不稳定。

后续还需要在 related work 或 method discussion 中进一步强调：创新点不是“adaptive trust region”本身，而是“由 contact consistency/stability signal 驱动的 adaptive trust-region control”。

### 3. 论文目前更适合走“诚实 scoped claim”路线

今天多处把强结论降级，说明当前证据最适合支持以下类型的主张：

- 在本实验设置下，FTC-triggered stabilization 与 PPO 骨干结合能改善某些训练稳定性指标；
- 对高并行设置，TC-PPO 提供了比无干预版本更可控的稳定机制；
- 对具体基线如 TD-MPC2，只能报告复现实验设定下的结果与差异，不能泛化为全面优越；
- 对敏感性分析，若缺少 pairwise 检验，就不能宣称显著性。

这会降低论文气势，但提高可信度。

### 4. Algorithm 1 的可复现性比公式更容易影响审稿观感

今天的算法重写很关键。对于机器人学习论文，审稿人往往会问：

- 触发频率是多少？
- 是否有 hysteresis？
- 激活后持续多久？
- 如何避免阈值附近振荡？
- 是否每个 update 都检查？
- 状态如何复位？

加入 `stabilization_active`、`\Delta_{check}`、`W` 后，算法从“概念性机制”变成更接近可实现的训练流程。这对公开仓库和后续复现实验也有直接帮助。
