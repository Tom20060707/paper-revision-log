## 新洞察

### 1. 图表问题不是“美观”这么简单，而是可信度问题

今天最大的洞察是，论文图表质量包含两层：

1. **视觉表达是否清晰**  
   如 legend 是否遮挡、颜色是否清晰、面板间距是否合理、注释是否重叠。

2. **数据是否真实可追溯**  
   如果图表使用了无法在 CSV / JSON 中追溯的数值，即使看起来像顶会风格，也不能用于投稿或开源。

因此后续论文图表原则应是：

> 先数据可复现，再视觉美化；不能用“示意性图”伪装成实验结果。

### 2. [[FTC Metric]] 的叙事应突出“early warning / intervention”而非普通性能提升

从 Figure 5 和 collapse 曲线的重构看，TC-PPO 最有力的故事不是单纯分数更高，而是：

- PPO 在某些 contact-rich / manipulation 任务中出现灾难性 collapse；
- [[FTC Metric]] 能作为 gait/contact temporal irregularity 的早期信号；
- 当 FTC 低于阈值，[[TC-PPO Algorithm]] 通过调节 entropy / KL 使更新更保守；
- 这种机制避免 reward 掉到 −9028 附近的灾难性区域。

这使论文叙事更接近“稳定性机制”而非普通 benchmark 刷分。

### 3. Off-chart 表达比强行同轴绘制更清楚

PPO collapse 的 −9028 与 TC-PPO 的 13–84 或 839–2401 差距过大。若强行放在同一 y 轴，会导致正值区间被压扁。今天采用 off-chart 注释方式，实际更符合顶会图表处理极端异常值的做法：

- 主图聚焦可比较的正值区域；
- collapse 数值用醒目标注呈现；
- 不牺牲可读性。

### 4. 在线 LaTeX 编译必须把“源码 + 图 + bib + 样式”视为一个项目

Texpage 空白图问题说明：本机可编译不代表在线可编译。在线平台无法访问：

- 本机绝对路径；
- Desktop 文件；
- 任何未上传 PDF / PNG / CSV；
- 本地 Python 生成流程。

因此面向投稿或 Texpage 的打包原则是：

- `.tex` 里只使用相对路径；
- 所有 `\includegraphics` 文件必须在项目 zip 内；
- 文件名避免空格和中文；
- 主入口统一命名为 `main.tex`；
- 若图由脚本生成，投稿包至少提供最终 PDF 图；开源包再提供脚本和数据。

### 5. 引用核查是必要步骤，尤其是 2025–2026 新论文

今天发现 FastTD3 和 PPO-BR 的引用曾出现错误：

- `arXiv:2506.18088` 被误认为 FastTD3，但实际是 RoboTwin 2.0；
- Hansen 版本的 “FastTD3: Simple, Fast, and Capable Reinforcement Learning for Continuous Control” 未确认存在；
- 正确 FastTD3 为 Seo et al.：
  - arXiv:2505.22642
  - “FastTD3: Simple, Fast, and Capable Reinforcement Learning for Humanoid Control”
  - Authors: Younggyo Seo, Carmelo Sferrazza, Haoran Geng, Michal Nauman, Zhao-Heng Yin, Pieter Abbeel
- `PPO-BR` 的 arXiv ID `2505.17714` 真实，但原标题“Behavior Regularization for Adaptive PPO in Humanoid Control”不准确，应修正为真实标题方向：“Dual-Signal Entropy-Reward Adaptation for Trust Region Policy Optimization”。

这说明后续所有新增引用必须经过：
- arXiv ID；
- title；
- authors；
- year；
- venue；
五项一致性核查。
