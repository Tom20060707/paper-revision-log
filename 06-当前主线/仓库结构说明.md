# 仓库结构说明

## 为什么要调整结构

旧仓库结构是在 4 月 16–27 的 TC-PPO 阶段建立的，当时关注点是：

- FTC 指标；
- Mode A/B/C；
- adaptive threshold；
- soft gating；
- PBRS/Kitchen 边界。

但 4 月 28–30 之后，论文主线已经明显上移：

```text
TC-PPO / FTC
→ CTS-PPO
→ CSC 总框架
→ 接触域一致性与稳定性研究
```

因此 README 和总览必须以当前主线为入口，旧目录只作为历史材料。

### `03-行动清单/`

当前后续任务。应优先服务“接触域一致性与稳定性”主线。

### `04-每日研究日志/`

日更日志镜像。内容来自 Obsidian 自动同步，偏当天记录。

最完整的 AI 对话重整区。已覆盖 `2026-02-15` 到 `2026-04-30` 的 52 篇按日摘要。

### `06-当前主线/`

当前最重要的仓库入口，负责解释论文现在到底收敛到什么方向。

## 对旧名词的处理

仓库不强行删除旧名词，因为它们是研究演化过程的一部分。

但阅读时应理解：

- `TC-PPO`：早期算法命名和实验记录入口；
- `FTC`：早期足部/肢体接触一致性指标；
- `CTS`：接触时序稳定性阶段；
- `CSC`：当前更一般的接触面一致性框架；
- `接触域一致性与稳定性`：当前论文主命题。
